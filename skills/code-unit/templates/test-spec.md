## 通用测试原则(兜底)

### 测试金字塔
- 大量单元测试(本技能专注)
- 适量集成测试
- 少量 E2E/系统测试

### 单元测试定义
- 测一个**单元**(函数/方法/类)
- 不依赖外部系统(数据库/网络/文件系统/外部服务),必要时用 mock
- 运行快(<1s)
- 独立运行,顺序无关

### AAA 结构
```
// Arrange: 准备数据
const input = ...
// Act: 调用被测单元
const result = functionUnderTest(input)
// Assert: 断言结果
expect(result).toBe(expected)
```

### 命名规范
- 测试文件:`<被测文件>.test.<ext>` / `<被测文件>_test.<ext>` / `<被测文件>Test.<ext>` / `<被测文件>.spec.<ext>`(按项目约定)
- 测试函数:`test_<单元>_<场景>_<期望>`(语言/框架不同可能不同)

### Mock 原则
- ✅ Mock 外部依赖(数据库、网络、第三方服务、文件系统、时间)
- ❌ 不 Mock 被测对象
- ❌ 不 Mock 简单的数据类
- ❌ 不通过 mock 验证实现细节

### 覆盖率
- 核心业务逻辑:高覆盖(>80%)
- 工具/配置代码:适度覆盖
- 第三方包装:豁免
- 覆盖率是手段不是目的,**关键路径覆盖**比数字更重要

### 测试即文档
- 阅读测试即可理解被测代码的契约
- 命名要表达意图
- 一个测试一个断言意图
- 失败时输出能直接定位问题

### 边界与异常
- 测空(空数组/空字符串/null/undefined)
- 测零/负数/极值
- 测超长输入
- 测 Unicode/特殊字符
- 测并发(若涉及)
- 测超时(若涉及)
- 测依赖不可用(若涉及)
