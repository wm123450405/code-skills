# 设计权衡笔记 — REQ-00041

## 关键设计问题

### Q1: references/ 目录放置方式
- **候选 A**:每个技能独立 references/,语言文档各自维护
- **候选 B**:共享 references/ 目录,所有技能引用同一份
- **选定 A**:每个技能独立 references/
- **否决 B 理由**:各技能的语言差异内容不同(code-design 关注项目结构,code-it 关注构建/测试命令),共享会导致文件过大且难以维护

### Q2: SKILL.md 简化程度
- **候选 A**:激进简化(仅步骤编号+标题)
- **候选 B**:适度简化(保留关键决策点)
- **候选 C**:保守简化(仅移动明确的语言差异)
- **选定 A**:激进简化
- **否决 B/C 理由**:用户明确选择"激进简化",目标是最大化 token 减少

### Q3: 语言检测时机
- **候选 A**:在 code-design 步骤 5 中检测
- **候选 B**:在 code-version 中统一检测
- **选定 A**
- **否决 B 理由**:code-version 是版本管理,不应承担语言检测职责;code-design 步骤 5 已有"探索项目结构"职责,自然扩展

### Q4: 语言标签传递方式
- **候选 A**:通过 design/<REQ>/RESULT.md 模块表"语言"列显式传递
- **候选 B**:下游技能各自重新检测
- **选定 A**:显式传递
- **否决 B 理由**:重复检测浪费 token,且可能导致不一致

### Q5: 是否删除原 SKILL.md 内容
- **候选 A**:移到 references 后删除原内容
- **候选 B**:移到 references 后保留原内容(注释)
- **选定 A**:删除原内容
- **否决 B 理由**:保留注释增加 token 消耗,与目标相悖;references 已完整保留细节
