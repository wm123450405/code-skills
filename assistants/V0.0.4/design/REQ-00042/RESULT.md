## 4. 关键流程

### 4.1 编码原则生效流程

```
code-it 步骤 8(实施开发)
  → 读取 references/common.md §5 "通用编码原则"
  → 命中新增条款"代码注释不引用内部追踪编号"
  → 命中新增子节"追踪编号禁用规则"(含正反面示例)
  → AI 在编写代码时自动遵循:
    - 注释中不出现 REQ-NNNNN / BUG-NNNNN / TASK-* 字面
    - 使用功能描述替代编号
    - 不写入任务信息(标题/状态/依赖)
```

### 4.2 模板约束生效流程

```

  → AI 在 RESULT.md 中记录"关键逻辑"时,不将编号写入代码示例
```

## 5. 方案选型

### 决策 1: 在 references/common.md 中落地规则

- **选择**:在 `references/common.md` §5 中新增条款和子节
- **备选**:在 `guidelines/coding-style.md` 中新增独立条款
- **选择理由**:
  - `references/common.md` §5 是 code-it 步骤 8 实施开发时**始终加载**的编码原则
  - 修改一处即可生效,不需要在多个文件中同步
  - `guidelines/` 目录目前作为预留,尚未被 code-it 正式加载
- **权衡**:若后续需要将"代码注释不引用编号"提升为项目级通用规范,可通过 `code-rule` 追加到 `rules/` 中
