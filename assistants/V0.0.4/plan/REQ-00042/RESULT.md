## 2. 逐文件详细设计

### 2.1 references/common.md §5

#### 当前状态(行 27-34)
```
### 通用编码原则
- 贴合项目现有风格
- 命名自解释,避免缩写
- 边界条件显式处理,不吞异常
- 关键路径埋点
- 代码内注释解释"为什么"
- 不引入未经评审的新依赖
- 每次只做本任务范围的事
```

#### 修改后状态
在 `- 代码内注释解释"为什么"` 之后插入 1 行:
```
- 代码注释不引用内部追踪编号:用功能描述替代 REQ-/BUG-/TASK- 编号
```

在 `### 审查改修任务特殊规则` 之前插入新子节:
```
### 追踪编号禁用规则
`/code-it` 产出的代码中不得出现 REQ-NNNNN / BUG-NNNNN / TASK-* 格式的编号。
代码注释应使用功能梗概替代编号:
- ✅ `// 用户登录:验证用户名密码,返回 JWT token`
- ❌ `// REQ-00042: 实现用户登录`
- ✅ `// 修复:空用户名导致 NullPointerException`
- ❌ `// BUG-00001: 修复空指针`
此约束不覆盖 commit message 和 `./assistants/` 工作产物。
```

#### 修改算法
1. 定位 `### 通用编码原则` 下的 `- 代码内注释解释"为什么"` 行
2. 在下一行插入新条款
3. 定位 `### 审查改修任务特殊规则` 行
4. 在其上一行(空行后)插入完整的"追踪编号禁用规则"子节(含标题 + 说明 + 4 个示例 + 约束说明)
