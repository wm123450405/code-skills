## 3. 关键设计决策

### 决策 D-1:硬约束锚点 = §"不要做的事" 小节末尾

- **候选位置对比**:

  - **§"不要做的事"小节末尾(选)**:沿用 BUG-00001 同类范式,语义最清晰
  - 新增 §"范围外"小节(拒):净增行数最多,违反 NFR-3 最小化变更

### 决策 D-2:对偶引用 `code-design`,但不修改 `code-design` SKILL.md

- **引用方式**:在新增条目中**显式**对偶引用"`code-design`(由其结合项目实际状况给出方案)"

### 决策 D-3:不引入"哪些需求需技术选型"判据表

- 沿用用户原始输入"**没有**必要进行技术选型的需求**无需**分析技术选型选项"
- `code-require` 端**只**做"不提技术选型"的硬约束,**不**给出反向判据

### 决策 D-4:不引入"是否需要技术选型"用户偏好

- `code-require` 步骤 7A 澄清**不**新增"本需求是否需要技术选型"问路
- 用户已显式回答"无需分析",无需再问(NFR-8 锁定)

### 决策 D-5:不修改 templates/requirements.md

- 模板**不**追加"不含技术选型"提示(NFR-2 零规范变更沿用 REQ-00032)
