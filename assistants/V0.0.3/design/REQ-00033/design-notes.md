## 2. 措辞候选(决策 D-1 锁定)

> `code-require` **不**涉及技术选型 / 技术栈 / 技术方案的确定;本技能只确定功能点(FR / NFR / 页面 / 交互 / 数据 / 边界);技术选型 / 架构风格 / 接口风格 / 数据模型(库表层) / 部署形态归 `code-design`(由其结合项目实际状况给出方案);没有**必要**进行技术选型的需求,**无需**在 `code-require` 阶段分析技术选型选项。

候选 2(简短式,拒):
> `code-require` 输出 = 需求提示词文档(**不**含技术选型 / 技术栈 / 技术方案);技术选型归 `code-design`。

候选 3(列表式,拒):列表展开 5 大类,行数过多(违反 NFR-3 净增 +2~+4 行)
