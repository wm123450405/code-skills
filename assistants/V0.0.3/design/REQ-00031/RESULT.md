## 1. 设计概述

本需求对 `code-plan` / `code-it` / `code-unit` / `code-auto` 四个元技能做**职责边界的硬切分**,并修订 1 个模板:

- **`code-plan` 任务粒度原则修订**——"完成定义"显式内化"编译/运行成功",任务类型移除 `测试`,任务"测试状态"枚举由 6 个收窄为 2 个
- **`code-it` 职责文档显式声明**——"## 目标"追加"不含单元测试"声明,与"## 工作流程"步骤 9-12 既有"测试"语义对齐
- **`code-unit` 职责文档显式声明**——"## 目标"追加"独立、可选"声明,既有的步骤 0-15 字节级保留
- **`code-auto` 任务循环步骤 4.b 修订**——"code-unit 是否调用"判定由原"按需调用"改为"恒等跳过";屏幕日志格式字节级保留
- **`templates/plan.md` 同步收窄**——任务类型移除 `测试`,测试状态字段收窄为 2 个

**核心架构思路**:**职责分离**(plan / it / unit 三技能独立清晰)+ **强制内化**(编译/运行下沉到每个任务)+ **可选外移**(单元测试由项目/用户按需自调),避免既有"职责混淆 + 冗余检测任务"双重痛点。

**新增/复用/修改范围**:5 个文件修改(4 SKILL.md + 1 templates),0 个文件新增;11 个 `code-*` 技能中 7 个 SKILL.md 字节级保留(code-require / code-design / code-check / code-dashboard / code-publish / code-version / code-fix 等);既有 11 个 REQ 的 `plan/PLAN.md` 0 改。

## 3. 设计目标与非目标

### 3.1 设计目标(对应上游 NFR-1)

- **目标 1**:`code-plan` 任务"完成定义"显式含"编译/运行成功";不再产出独立"编译运行检测"任务
- **目标 2**:`code-plan` 不再产出"测试"类型任务;任务"测试状态"统一收窄为 `不适用`(初始值)
- **目标 3**:`code-it` = 编码 + 编译/运行成功;`code-unit` = 单元测试(独立、可选);`code-plan` = 只规划编码部分 — 三者功能独立、界限清晰
- **目标 4**:`code-auto` 任务循环**永不再调** `code-unit`(由目标 2 + 步骤 4.b 修订共同保证)
- **目标 5**:`templates/plan.md` 同步收窄,任务清单自动同步(由 `code-plan` 步骤 15A 沿用)

### 3.2 非目标(强约束)

- **不**改 `code-unit` 内部行为(步骤 1-15 字节级保留)
- **不**改 `code-check` 评审清单(8.7 测试质量维度字节级保留)
- **不**重写 4 个被改 SKILL.md 的全部内容,只**追加**文档头 1 行声明 + 修订指定章节
- **不**追溯重写既有 11 个 REQ 的 `plan/PLAN.md`(NFR-2)

## 8. 风险与缓解

| 风险 | 可能性 | 影响 | 缓解 |
| --- | --- | --- | --- |
| 既有 11 个 REQ 的 `plan/PLAN.md` 任务"测试状态"列出现"未编写 / 已编写 / 已运行-通过 / 已运行-失败" | 高 | 既有任务"测试状态"列与新规则的"2 选 1"不兼容 | NFR-2 沿用:**不**追溯重写;新拆分 REQ(REQ-00031+)应用新规则 |
| 既有任务的"任务类型"列含 `测试` | 中 | 既有"测试"类型任务与新规则的"5 选 1"不兼容 | NFR-2 沿用:不追溯重写;code-check §8.7 字节级沿用 |
| `code-auto` 步骤 4.b 改成"恒等跳过"后,`code-unit` 屏幕日志**(跳过,无需测试)** 出现频率显著升高 | 高 | 屏幕日志视觉冗余 | INV-10 字节级保留:屏幕日志格式**不**改(为后续若 `code-plan` 重新启用测试规划时,可**字节级**还原) |
| `code-unit` 文档头追加"独立、可选"声明可能被误解为"code-unit 即将弃用" | 低 | 用户/AI 误判 | 声明明确"由项目/用户按需在 code-it 之后**自行**调用"(正交语义) |
| 5 个被改文件不在同一目录,code-it 串行实施时不同任务的"涉及文件"分散 | 中 | 任务粒度不均 | T-001 ~ T-005 每任务 1 文件;文件分散不构成障碍 |
| `code-auto` 子技能零修改契约(NFR-4)被破坏 | 低 | 4 个被改 SKILL.md 实际是被 code-auto 间接修改 | 校验:`code-auto` 自身 SKILL.md **不**在本需求的 4 个被改 SKILL.md 之列(本需求改的是 4 个**子**技能,不改编排者) |
