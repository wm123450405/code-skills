## 候选方案权衡(已锁定)

### 方案 A:用 Python 脚本写 .docx
- 优点:用户原文期望;真正支持二进制格式
- 缺点:本仓库 CLAUDE.md 强约束"不假设任何工具链",引入 Python 违反 `dependency-conventions`
- 决策:**不采纳**(本需求 0 新增依赖;二进制格式留作 follow-up)

### 方案 B:用 Node 脚本写 .docx
- 优点:同 A
- 缺点:同 A(本仓库无 Node 工具链)
- 决策:**不采纳**

### 方案 C:占位符替换 + Write 工具(可文本化格式)(本需求采纳)
- 优点:技术完全可行;零新增依赖;尊重用户原文意图(可文本化格式)
- 缺点:二进制格式(.docx/.xlsx/.pdf)本需求不实现,需 follow-up
- 决策:**采纳**
