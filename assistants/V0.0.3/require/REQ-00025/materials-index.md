## 用户原始输入
- **类型**:自然语言描述(无材料附件)
- **完整文本**(本轮用户输入):
  > 优化项目中所有技能对各类编号格式的判定逻辑,生成逻辑保持不变,但允许用户使用自定义的编码规则,只保持编号前缀的规则。比如允许用户使用 REQ-V0.0.1.001 类似的格式定义需求编号,不在需求编号中使用强制性较高的正则规则,可以要求前缀部分,比如需求编号前缀固定为 REQ-。方便用户使用其他平台登记需求、任务、缺陷后,由其他平台将内容到处到项目中再通过技能进行代码编写,此时需求、任务、缺陷编号皆由第三方平台生成。

## 关键摘录
- **本需求核心**:放宽"编号正则强约束",**仅保留前缀**(REQ-/BUG-/TASK-)
- **生成逻辑保持不变**:`code-require` / `code-plan` 等生成端继续按 5 位纯数字生成
- **接收逻辑放宽**:解析端接受任意后缀(如 `JIRA-123` / `REQ-V0.0.1.001`)
- **典型应用场景**:第三方平台(Jira / 禅道 / Redmine / GitLab)登记需求/任务/缺陷,导出到本项目后由 `code-skills` 技能接管开发
