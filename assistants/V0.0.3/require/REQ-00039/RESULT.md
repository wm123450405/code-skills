## 1. 需求概述

**优化** `/code-it`、`/code-check` 等技能在涉及"代码行数限制"判定时的语义:仅统计**实际代码逻辑行数**,排除 4 类非逻辑行(注释 / 说明 / 空行 / 格式化换行)。

**核心机制**:
- **逻辑行计算函数**(FR-1):明确定义"逻辑行 = 总行 - 空行 - 注释行";4 类排除项字面定义
- **工具集成**(FR-2):首选 `tokei`(全语言,200+)/ 备选 `cloc`(50+)/ 回退启发式(无依赖,~95% 精度)
- **`code-it` 步骤 8 末尾 `calcLogicLoc` 子步骤**(FR-3):编码完成后收集逻辑行 metadata,写入 `code/<TASK-...>/RESULT.md`
- **`code-check` 步骤 8 评审新增"代码行数超标"发现**(FR-4):评审时读 metadata,超阈值派生发现(P2 性能 / P3 可维护性 子项)
- **阈值配置**(FR-5):默认单文件 ≤ 500 行(总规模)/ ≤ 200 行(新增),用户可配置

涉及 2 SKILL.md(`code-it` + `code-check`)+ 1 模板(`code-it/templates/RESULT.md` 改造),共约 3 个文件,净变化约 +80 ~ +150 行(扩展,不删除)。

## 4. 功能需求(FR)

### FR-1:逻辑行计算函数(共享定义)

- **定义**:`逻辑行 = 总行 - 空行 - 注释行`
- **4 类排除项**:
 1. **注释行**:
 - 行级注释:`//` (JS/TS/Go/Java/Rust) / `#` (Python/Ruby) / `--` (SQL/Lua) / `<!-- -->` (HTML/Markdown)
 - 块级注释按行展开:`/* */` / `""" """`(Python docstring) / `''' '''`(Python docstring)
 2. **说明行**:文档字符串(docstring)/ 注释块内的"说明"文本(沿用注释行)
 3. **空行**:全空白字符 / 仅含换行符的行
 4. **格式化换行**:prettier / black / gofmt 等自动格式化产生的换行
 - 判定规则:若该行有代码标识符 / 运算符,**计**为逻辑行;若只有括号 / 逗号(纯格式化换行),**不**计
 - 简化:依赖工具(tokei / cloc)处理,工具**不**计纯格式化换行
- **共享位置**:`code-check` + `code-it` 共用同一逻辑行计算函数(避免重复定义)
- **不触发**:`./assistants/rules/dashboard-conventions.md` §规则 1(本需求**不**新增列)

### FR-2:工具集成 — tokei / cloc / 启发式回退

- **首选工具**:`tokei`(Rust 编写,支持 200+ 语言)
- **备选工具**:`cloc`(Perl 编写,支持 50+ 语言)
- **回退**:启发式(无依赖,5 主流语言,~95% 精度)
- **检测机制**:
 ```
 1. Bash: tokei --version → 命中 → 使用 tokei
 2. 否则 Bash: cloc --version → 命中 → 使用 cloc
 3. 都不存在 → 屏显 ⚠ + 回退启发式
 ```
- **tokei 输出**:`tokei <file> --output json` → `{ "<lang>": { "code": N, "comments": N, "blanks": N } }`
- **cloc 输出**:`cloc <file> --json` → 同样 JSON 格式
- **启发式算法**:
 1. 读文件全部行
 2. 过滤空行(`^\s*$`)
 3. 过滤注释行(正则,按语言):
 - Python: `^\s*(#|""" )`
 - JS/TS/Go/Java/Rust: `^\s*(//|/\*|\*)`
 - Markdown: `^\s*<!--`
 4. 剩余行数 = 逻辑行
- **启发式精度**:~95%(块级注释按行展开,字符串内注释符号不特殊处理)
- **安装责任**:本技能**不**负责安装 tokei / cloc,仅检测

### FR-3:`code-it` 步骤 8 末尾追加 `calcLogicLoc` 子步骤

- **位置**:`plugins/code-skills/skills/code-it/SKILL.md` §"## 步骤 8 实施开发"**末尾**追加子步骤
- **触发条件**:`code-it` 步骤 1 判定为任务分支(`^TASK-REQ-\d{5}-\d{5}$` 5+5 位嵌套式);缺陷分支(`^TASK-BUG-...`)**不**触达
- **算法**:
 ```
 function calcLogicLoc(taskNum, timestamp):
 1. git diff --numstat HEAD~1 -- <task 涉及文件> → 变更文件列表
 2. 对每个变更文件:
 - tokei / cloc / 启发式 → 读取"修改后逻辑行"
 - 若修改前可获取(git show HEAD~1:<file>) → 计算"新增逻辑行" = 修改后 - 修改前
 - 若新建文件 → "新增逻辑行" = 修改后
 3. 累计"本任务逻辑行新增" = Σ 新增逻辑行
 4. 累计"本任务逻辑行总规模" = Σ 修改后文件逻辑行
 5. 写入 code/<task>/RESULT.md 新增"## 逻辑行统计"小节
 6. 失败 → 屏显 ⚠ + 跳过(不阻断 code-it)
 ```
- **输出格式**(`code/<task>/RESULT.md` 新增"## 逻辑行统计"小节):
 ```
 ## 逻辑行统计(由 code-it 内化)

 | 文件 | 逻辑行(新增) | 逻辑行(总规模) | 检测方式 |
 | --- | --- | --- | --- |
 | src/foo.ts | 25 | 80 | tokei |
 | src/bar.py | 10 | 30 | tokei |
 | **本任务汇总** | **35** | **110** | — |
 ```
- **屏显契约**(沿用既有 `code-it` 步骤 8a.4 / 8.5.3 模式):
 ```
 === code-it 逻辑行统计(步骤 8 末尾)===
 任务:<任务编码>
 变更文件:N 个
 逻辑行(新增):<Σ>
 逻辑行(总规模):<Σ>
 检测方式:tokei / cloc / 启发式
 ```
- **不修改**:`code-it` 既有步骤 8 子步骤结构
- **不修改**:`code-it` frontmatter / 既有"## 工作流程"小节 / "## 不要做的事"小节

### FR-4:`code-check` 步骤 8 评审新增"代码行数超标"发现

- **位置**:`plugins/code-skills/skills/code-check/SKILL.md` §"## 步骤 8 逐任务评审" + §"## 评审维度速查表"(line 574-589)
- **新增子步骤**:§"## 步骤 8.13 — 代码行数超标检查"
- **算法**:
 ```
 1. 读 code/<task>/RESULT.md 的"## 逻辑行统计"小节
 2. 读阈值配置(默认 500 / 200,可由 FR-5 配置覆盖)
 3. 对每个变更文件:
 - 若"逻辑行(总规模)" > 阈值 → 派生"代码行数超标"发现
 - 若"逻辑行(新增)" > 阈值 → 派生"代码行数超标"发现
 4. 派生发现的级别:
 - 超阈值 ≤ 10% → 可选
 - 超阈值 > 10% 且 ≤ 50% → 建议改
 - 超阈值 > 50% → 必须改
 ```
- **评审维度速查表新增**:
 | 优先级 | 维度 | 不通过时一般标 |
 | --- | --- | --- |
 | P3 | 代码行数超标 | 可选 / 建议改 / 必须改(按超标百分比) |
- **发现描述格式**:
 ```
 [代码行数超标] <file>
 - 维度:P3(可维护性)
 - 逻辑行(总规模):<N>
 - 阈值:<M>(超 <P>%)
 - 建议改:<具体拆分建议 / 抽取公共函数 / 重命名空间 等>
 ```
- **不修改**:`code-check` 既有步骤 8.1 ~ 8.12 子步骤(字节级保留)
- **不修改**:`code-check` 评审维度速查表既有 12 维度(字节级保留,仅追加第 13 维度)

## 9. 边界与异常

- **E-1**:tokei 不存在 + cloc 不存在 → 屏显 `⚠` + 启发式回退
- **E-2**:git diff 失败(非 git 仓库)→ 屏显 `⚠ 非 git 仓库,无法获取变更文件列表;本次跳过 calcLogicLoc`
- **E-3**:变更文件无法访问(权限/不存在)→ 跳过该文件,屏显警告
- **E-4**:单文件过大(> 10MB)→ 跳过该文件,屏显警告(性能)
- **E-5**:无 `code/<task>/RESULT.md` 的"## 逻辑行统计"小节(老任务 / REQ-00039 之前)→ `code-check` 步骤 8.13 跳过该任务,不派生发现
- **E-6**:阈值配置缺失 → 默认 500 / 200
- **E-7**:缺陷分支(`^TASK-BUG-...`)→ **不**触达 `calcLogicLoc`(沿用 NFR-8)
- **E-8**:`code-auto` 上下文 → 沿用既有 NFR-7 退出码契约
