## 1. 核心痛点(用户原始输入)

**关键短语**:

- "`/code-it`、/code-check 等技能" → 涉及 2 个主技能,可能扩展更多

## 3. 优化方案(用户 1 轮澄清后定稿)

### 3.1 逻辑行计算函数(FR-1)

**核心定义**:逻辑行 = 总行数 - 空行 - 注释行(纯白字符行算空行)

**4 类排除项**(沿用用户原始输入):
- **注释行**:行级注释 + 块级注释按行展开
- **说明行**:文档字符串(docstring)/ 注释块内的"说明"文本(沿用注释行)
- **空行**:全空白字符 / 仅含换行符的行
- **格式化换行**:prettier / black / gofmt 等自动格式化产生的"逻辑换行"
 - 例:`const result = a + b + c;`(单行) vs `const result =` + `  a +` + `  b +` + `  c;`(格式化换行)
 - 判定:若该行有代码标识符或运算符,**计**为逻辑行(因为它承载代码逻辑);若该行只有括号 / 逗号,**不**计
 - 简化:依赖工具(tokei/cloc)处理,工具**不**计纯格式化换行

### 3.2 工具集成(FR-2,沿用 tokei/cloc)

**检测机制**:
```
1. Bash: tokei --version → 命中 → 使用 tokei
2. 否则 Bash: cloc --version → 命中 → 使用 cloc
3. 都不存在 → 屏显 ⚠ 警告 + 回退到启发式
```

**tokei 输出格式**(JSON 模式 `--output json`):
```json
{
 "Python": {
 "code": 30, // ← 即"逻辑行"
 "comments": 40,
 "blanks": 20
 },
 ...
}
```

**cloc 输出格式**(JSON 模式 `--json`):
```json
{
 "Python": {
 "code": 30,
 "comment": 40,
 "blank": 20
 },
 ...
}
```

**启发式回退**(都不存在时):
1. 读文件全部行
2. 过滤空行(`^\s*$`)
3. 过滤注释行(正则,按语言):
 - Python: `^\s*#`
 - JS/TS/Java/Go: `^\s*//` 或 `^\s*\*` 或 `^\s*/\*`
 - Markdown: `^\s*<!--`
4. 剩余行数 = 逻辑行

**启发式精度**:
- 块级注释(docstring)按行展开,**不**特殊处理(精度略低)
- 字符串内的 `#` / `//` **不**特殊处理(精度略低)
- 接受 ~5% 误差(沿用既有"启发式精度"约定)

### 3.3 code-it 步骤 8 末尾追加 `calcLogicLoc` 子步骤(FR-3)

**触发条件**:
- `code-it` 步骤 1 判定为任务分支(`^TASK-REQ-\d{5}-\d{5}$` 5+5 位嵌套式)
- 缺陷分支(`^TASK-BUG-...`)**不**触达(沿用既有步骤 8a 守卫规则)

**算法**:
```
function calcLogicLoc(taskNum, timestamp):
 1. git diff --numstat HEAD~1 -- <task 涉及文件> → 变更文件列表
 2. 对每个变更文件:
 - tokei/cloc/启发式 → 读取逻辑行
 - 计算 "新增逻辑行" = 修改后 - 修改前(若新建文件 = 修改后)
 3. 累计 "本任务逻辑行新增" = Σ 新增逻辑行
 4. 累计 "本任务逻辑行总规模" = Σ 修改后文件逻辑行
 5. 写入 code/<task>/RESULT.md 新增"## 逻辑行统计"小节:
 - 单文件逻辑行(新增 / 总规模)明细
 - 本任务汇总
 6. 失败 → 屏显 ⚠ + 跳过(不阻断 code-it)
```

**输出格式**(`code/<task>/RESULT.md`):
```
## 逻辑行统计(由 code-it 内化)

| 文件 | 逻辑行(新增) | 逻辑行(总规模) | 检测方式 |
| --- | --- | --- | --- |
| src/foo.ts | 25 | 80 | tokei |
| src/bar.py | 10 | 30 | tokei |
| **本任务汇总** | **35** | **110** | — |
```

### 3.4 code-check 步骤 8 评审新增"代码行数超标"发现(FR-4)

**触发条件**:
- `code-check` 步骤 8 逐任务评审
- 读 `code/<task>/RESULT.md` 的"## 逻辑行统计"小节
- 若"单文件逻辑行总规模 > 阈值"或"单文件逻辑行新增 > 阈值"→ 派生"代码行数超标"发现

**阈值默认值**:
- 单文件逻辑行总规模 ≤ 500 行(沿用既有"代码行数"经验值)
- 单文件逻辑行新增 ≤ 200 行(单任务不应大幅增加单文件规模)
- 阈值由用户配置(`requirements/.../RESULT.md` §"## 阈值配置"小节),默认 500 / 200

**发现描述**:
- 维度:P3(可维护性)或 P2(性能)子项
- 级别:超阈值 ≤ 10% 标"可选";超阈值 > 10% 标"建议改";超阈值 > 50% 标"必须改"
- 文案:`⚠ <file> 逻辑行 <N> 超过阈值 <M>(超 <P>%)`

### 3.5 阈值配置(FR-5,可选)

**位置**:`assistants/<版本号>/requirements/.../RESULT.md` 新增"## 阈值配置"小节(若需)

**配置字段**:
- `单文件逻辑行总规模阈值:<N>(默认 500)`
- `单文件逻辑行新增阈值:<N>(默认 200)`

**默认值**:无配置时使用 500 / 200

## 4. 边界与异常

### 4.1 退化路径

- **tokei 不存在 + cloc 不存在** → 屏显 `⚠ 建议安装 tokei/cloc 以获得准确逻辑行统计;本次回退到启发式`,使用启发式算法
- **git diff 失败(非 git 仓库)** → 屏显 `⚠ 非 git 仓库,无法获取变更文件列表;本次跳过 calcLogicLoc`
- **变更文件无法访问(权限/不存在)** → 跳过该文件,屏显警告
- **单文件过大(> 10MB)** → 跳过该文件,屏显警告(性能)

### 4.2 性能

- tokei/cloc 调一次耗时:< 1 秒(100 文件典型)
- 启发式耗时:< 2 秒(100 文件典型)
- 总体耗时:< 3 秒,可接受

### 4.3 与既有约束的兼容

- **不修改 frontmatter**:`code-check/SKILL.md` + `code-it/SKILL.md` L1-3 字节级保留
- **不修改既有"## 工作流程"小节**:沿用既有结构,仅在步骤 8 末尾追加
- **不修改"## 不要做的事"小节**:沿用既有 N-1 ~ N-N
- **不触发 `AskUserQuestion`**:本技能沿用 NFR-3 强约束,行数计算全部自动
- **不新增 CLI 参数**:阈值沿用默认 500 / 200(无 `--max-loc` 等)

## 5. 候选方案权衡

| 维度 | 启发式(无 tokei) | tokei(全语言) | cloc(全语言) |
| --- | --- | --- | --- |
| 精度 | ~95% | ~99% | ~99% |
| 安装复杂度 | 0(无依赖) | 中(Rust 工具链) | 中(Perl 工具链) |
| 支持语言 | 5 主流(本次限制) | 200+ | 50+ |
| 性能 | 中(读全文正则) | 快 | 中 |
| 块级注释 | 不支持(按行展开) | 支持 | 支持 |
| 字符串内注释符号 | 不支持 | 支持 | 支持 |

**最终选择**:tokei 优先 + cloc 备选 + 启发式回退(用户选 B 全语言)
