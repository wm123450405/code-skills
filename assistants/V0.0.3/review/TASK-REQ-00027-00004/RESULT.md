## 2. 问题清单

### 问题 F-1:步骤 4 状态推进表保留中间/非本技能终态行
- **位置**:`plugins/code-skills/skills/code-fix/SKILL.md` L249-261
- **类别**:设计符合度
- **严重程度**:必须改
- **依据**:
  - 规则:无强制条款(`skill-conventions` 仅约束 frontmatter)
  - 设计:`./assistants/V0.0.3/design/REQ-00027/RESULT.md` §2.2 / §2.4 决策 1
  - 详细设计:`./assistants/V0.0.3/plan/REQ-00027/RESULT.md` §2.1 模块 1 步骤 4 子节
  - 需求:`./assistants/V0.0.3/require/REQ-00027/RESULT.md` FR-3 + AC-1.1
  - 关联原任务:T-001 code/RESULT.md "9 个区段全部 Edit"
- **现象**(`code-fix/SKILL.md` L249-261):
  ```
  | 当前状态 | 候选目标状态 |
  | --- | --- |
  | (新建) | 报告 / 调查中 |
  | 报告 | 调查中 / 已关闭-非缺陷 / 已取消 |
  | 调查中 | 已关闭-非缺陷 / 阻塞 / 已取消 |
  | 修复规划中 | (由 `code-plan` 推进;本技能不推进此状态之后) |  ← 设计要求删除
  | 修复编码中 | (由 `code-it` 推进) |  ← 设计要求删除
  | 已修复-待验证 | (由 `code-it` 推进) |  ← 设计要求删除
  | 已修复-已验证 | (由 `code-check` 推进) |  ← 设计要求删除
  | 已关闭-非缺陷 | (终态) |
  | 已关闭-不修复 | (终态) |  ← 设计要求删除(本技能不参与)
  | 已取消 | (终态) |
  | 阻塞 | 解除阻塞(回到报告中合适状态) / 已取消 |
  ```
- **问题描述**:详细设计 §2.1 明确要求"候选目标状态表**改为**"——即从原 9 候选行收敛到 5 候选行(报告 / 调查中 / 已关闭-非缺陷 / 已取消 / 阻塞)。当前实现保留了"修复规划中 / 修复编码中 / 已修复-待验证 / 已修复-已验证 / 已关闭-不修复"等非本技能候选行,虽然加了"(由 X 推进)"说明,但**视觉上**仍是 11 行表,违反"5 个候选状态"字面要求;对用户而言会把"修复规划中"误认为可由本技能推进的候选目标。
- **建议改修**:
  ```
  | 当前状态 | 候选目标状态 |
  | --- | --- |
  | (新建) | 报告 / 调查中 |
  | 报告 | 调查中 / 已关闭-非缺陷 / 已取消 |
  | 调查中 | 已关闭-非缺陷 / 阻塞 / 已取消 |
  | 已关闭-非缺陷 | (终态) |
  | 已取消 | (终态) |
  | 阻塞 | 解除阻塞(回到报告中合适状态) / 已取消 |
  ```
  同时,删去"超出本技能范围"小节(原 L268-271),其内容已由步骤 5 / 步骤 9 / 步骤 10 的"典型修复流程"段承接。
- **替代方案**:在表前加注释"以下 5 行为本技能可推进的候选目标;其他中间/终态由 code-plan / code-it / code-check 推进" — 视觉上仍保留 11 行,**不推荐**。
