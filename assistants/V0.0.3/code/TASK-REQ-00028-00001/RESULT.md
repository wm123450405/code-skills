## 3. 关键决策与权衡

### 3.1 决策 1:本技能**不**做"反向查询"(从代码反向定位需求)

- **理由**:用户澄清记录 Q-1 已显式推迟到 v2
- **影响**:本技能仅支持"需求 → 代码"单向查询;反向场景留作后续
- **优点**:保持职责单一,降低复杂度
- **缺点**:新成员若想"知道这段代码对应哪条需求"无解

### 3.2 决策 2:关键字匹配**不**支持正则(仅字面 + 忽略大小写)

- **理由**:用户澄清记录 Q-2 已显式推迟到 v2
- **影响**:用户输入 `code-(design|dashboard)` 不会被作为正则处理
- **优点**:实现简单,行为可预期
- **缺点**:高级用户无法利用正则表达力

### 3.3 决策 3:报告**不**支持"导出为 Markdown"

- **理由**:用户澄清记录 Q-3 已显式推迟到 v2
- **影响**:用户需要手动从终端复制到 `.md` 文件
- **优点**:保持"仅屏显"契约严格性
- **缺点**:不便于长期归档
