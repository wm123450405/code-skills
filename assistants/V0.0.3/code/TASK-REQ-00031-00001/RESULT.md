- **D-1 任务完成定义内化编译/运行**:将"编译/运行成功"从"每任务 code-it 步骤 9-12 的隐式保证"提升为"任务完成定义的显式组成部分"。理由:用户原文"每个任务应该确保编译或运行成功" — 显式声明消除歧义。

- **D-4 双状态语义简化**:由"开发 ∧ 测试"两轴简化为"开发"单轴。理由:本仓库 markdown 元技能无测试框架;双轴语义对纯文档类任务是冗余。
