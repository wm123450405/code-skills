| T-1(步骤 7A 候选选项过滤)20 关键词 | T-2(步骤 8A NFR 过滤)14 关键词 |

**理由**:
- T-1 关键词集针对"用户问路选项",需覆盖"如何实现"问路的常见候选(框架/库/工具)

### 4.2 决策:`clarifications.md` 写入不同区段

**决策**:T-2 写入 `## NFR 延迟到 code-design 阶段(技术选型类)`,与 T-1 的 `## 延迟到 code-design 阶段(技术选型类)` **分门别类**。

**理由**:
- T-1 是"问路"性质的延迟(`AskUserQuestion` 候选选项)
- T-2 是"NFR 记录"性质的延迟(`RESULT.md` NFR 章节)
- `code-design` 阶段扫读时,先看"## NFR 延迟到"区段知道哪些 NFR 需要重写;再看"## 延迟到"区段知道哪些候选问题需要重新提问
- 两个区段语义独立,合并会导致 `code-design` 阶段识别困难
- 不引入新字段(沿用 `## 区段` 分组范式,符合 `skill-conventions §规则 2`)

**决策**:本子节显式引用"步骤 7A 末尾 + 步骤 615",形成完整技术选型过滤链。
