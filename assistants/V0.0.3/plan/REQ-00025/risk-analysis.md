# 风险分析 — REQ-00025
版本:V0.0.3

## 异常处理

- 异常路径 1:**后缀空**(`REQ-` / `BUG-` / `TASK-REQ--1`)
  - 描述:`match()` 返回 `null`
  - 处理:屏显 `⚠ 编号后缀为空: <input>` + 退出
  - 监控:屏显率(无性能监控)
  - 对应任务:TASK-REQ-00025-00001 (encoding-conventions)

- 异常路径 2:**后缀含非法字符**(`/` / `\` / `!` 等)
  - 描述:`match()` 返回 `null`
  - 处理:屏显 `⚠ 编号含非法字符: <char>` + 退出
  - 监控:屏显率
  - 对应任务:TASK-REQ-00025-00001

- 异常路径 3:**第三方平台前缀未登记**(`PROJ-123`)
  - 描述:前缀不匹配 4 类登记前缀(`REQ-` / `BUG-` / `TASK-REQ-` / `TASK-BUG-`)
  - 处理:屏显 `⚠ 前缀 <X>- 未登记` + 引导调 `code-rule`
  - 监控:屏显率
  - 对应任务:TASK-REQ-00025-00001 (§规则 1.5)

- 异常路径 4:**OS 文件系统拒绝**(`REQ-a/b` 在 Windows 路径层)
  - 描述:`mkdir` 失败
  - 处理:屏显 `⚠ 文件系统不支持字符: <char>` + 退出
  - 监控:OS 层拒绝率
  - 对应任务:无任务(沿用既有 OS 限制)

## 安全边界

- **路径遍历防护**:后缀字符集 `[A-Za-z0-9.\-_]+` 排除 `/` `\`,**不**解析 `..`(虽然 `..` 字符在字符集内,但 OS 文件系统层会拒绝)
- **注入风险**:新正则更严格(排除 `!@#$%^&*()` 等);OS 文件系统层会拒绝特殊字符
- **敏感数据处理**:N/A(本仓库无敏感数据)
- **审计日志**:
  - 屏显保留完整编号(便于审计)
  - 变更记录追加 1 行"编号格式放宽"
- **依据规范**:`encoding-conventions.md §规则 1.5`(本需求新增)
