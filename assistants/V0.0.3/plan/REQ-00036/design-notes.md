# 设计笔记 — REQ-00036(详设阶段)

更新时间:2026-06-16 17:33
版本:V0.0.3

## 关键设计问题清单(详设阶段细化)

1. **Q-1**:6 条清理规则的精确正则字符集怎么写?
2. **Q-2**:白名单例外是"独立白名单文件"还是"正则负向断言"?
3. **Q-3**:`code-fix` 技能对自身 `fix/` 目录的引用怎么豁免?
4. **Q-4**:任务拆分粒度 — 1 大任务 vs 3 小任务?
5. **Q-5**:改动后大段残缺的判定阈值是多少?
6. **Q-6**:AC 验证允许的例外数(AC-2 / AC-4)怎么定?

## 候选方案(对每个关键问题)

### Q-1 精确正则字符集

- **方案 A**:宽松正则(无负向断言)→ 命中多但有误伤
- **方案 B**:精确正则(带负向断言)→ 命中少但精准
- **方案 C**:两遍扫描(先宽松,后人工判)→ 折中

**选定:B(精确正则)**
- 理由:6 条规则各带负向断言(避开 `(本仓库主动产出)` / `(沿用 V0.0.X NFR-Y)` / `(既有 N 需求 NFR-5 ...)` 等白名单字面),一次扫描到位

### Q-2 白名单实现

- **方案 A**:独立白名单 JSON 文件(可被 AI 编辑)→ 灵活但增加配置面
- **方案 B**:正则负向断言(写死在规则里)→ 简单但难扩展

**选定:B(负向断言)**
- 理由:本轮 4 类白名单全是字符集层面的(无"逐文件豁免"需求);负向断言已足够

### Q-3 code-fix 豁免

- **方案 A**:正则负向断言(对 `code-fix/...` 跳过 R-5)
- **方案 B**:任务级特例(对 `code-fix` 技能整个目录跳过 R-5)
- **方案 C**:不清 R-5 命中,人工 review

**选定:A(正则负向断言 + 算法 0 范围过滤)**
- 理由:`code-fix` 自身 SKILL.md 中"R-1 ~ R-4 + R-6"仍要清(它可能也有段尾/回溯/决策等痕迹),只 R-5(退场文件名)豁免

### Q-4 任务拆分粒度

- **方案 A**:1 大任务(扫描 + 应用 + 验证合一)
- **方案 B**:3 任务(扫描 / 应用 / 验证分开)
- **方案 C**:2 任务(应用 + 验证,T-1 合并到 T-2)

**选定:B(3 任务)**
- 理由:T-1 产出的"文件表 + 命中计数"是 T-2 的输入,T-2 的 diff 是 T-3 的输入;3 步严格串行;每个任务 0.5 天内可完成

### Q-5 改动后残缺判定

- **方案 A**:固定阈值(> 30% 减少)
- **方案 B**:无阈值,人工判
- **方案 C**:0 阈值(任意改动都接受)

**选定:A(30% 阈值)**
- 理由:30% 是经验值(本仓库 SKILL.md 平均 ~ 500 行,30% = 150 行);若改动 > 150 行,大概率是"剥离了实质内容"

### Q-6 AC 例外数

- **方案 A**:0 例外(全清)
- **方案 B**:≤ 3 处例外(AC-2 / AC-4)
- **方案 C**:无上限

**选定:B(≤ 3 处)**
- 理由:与 `skill-conventions §规则 2` 的"合规性条款 / 跨版本规范继承引用"2 类白名单对齐;留 1 处余量

## 选定方案与理由汇总

**整体方案**:`--minimal` 路线,3 任务严格串行(扫描 / 应用 / 验证),6 条精确正则(带负向断言),1 次 commit,`code-fix` 技能 R-5 豁免。

**理由**:
1. 用户选 `--minimal` + 沿用 design 的设计目标
2. 6 条规则的负向断言已覆盖 4 类白名单,无需独立配置文件
3. 3 任务拆分让每步可独立验证(AC 验证在 T-3 集中跑)
4. 整体无新增模块 / 接口 / 数据结构 / 三方依赖(符合 `--minimal`)

## 备选方案被否决的理由(汇总)

- **1 大任务**:违反"0.5-2 天内可完成"原则
- **独立白名单文件**:增加配置面,本轮不必要
- **0 改动阈值**:风险大,易误伤实质内容
- **0 AC 例外**:可能误伤合规条款

## 实施要点(给 code-it 阶段)

- **T-1 扫描**:用 `Grep` 工具逐规则扫,产出 `code/<TASK>/work-log.md` + `RESULT.md`(含文件表 + 命中数)
- **T-2 应用**:对每个文件,用 `Edit` 工具跑 6 条规则;**不**用 `Write`(会丢失历史)
- **T-2 残缺回退**:对每个文件 `git diff --stat`,若行数减少 > 30% → `git checkout HEAD -- <file>` 回退
- **T-3 验证**:用 `Grep` 跑 8 条 AC;`/code-dashboard` 通过 `Bash` 调
- **T-3 提交**:1 次 `git commit`,commit message 包含 REQ-00036
