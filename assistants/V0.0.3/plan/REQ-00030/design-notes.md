## 2. 关键决策(详设阶段)

### 2.1 步骤 0b 自适应问路(对应 FR-1 + FR-2)

**完整决策树**(伪代码):
```
function adjustAskUserQuestionCount(req: ReqSummary, codeAutoCtx: bool): int {
  if codeAutoCtx:
    return 0  // code-auto 上下文永远 0 问

  // FR-1 判定:是否触发扩展性
  triggersExtensibility = (
    req.pendingThirdPartyDeps.length > 0          // 条件 1
    || req.predictedModuleCount >= 3               // 条件 2
    || /\b(多实现|抽象层|可替换|多套实现)\b/.test(req.frText)  // 条件 3
  )

  // FR-2 收紧:小需求 0 问
  if req.taskCount <= 1 && req.isSmall(req) && !triggersExtensibility:
    return 0  // 小需求 + 不触发扩展性 → 0 问

  if req.taskCount >= 6 || req.needsArchitectureSkeleton(req):
    return 2  // 大需求 → 2 问(整体 + 功能性)

  return 1  // 中等需求 / 触发扩展性 → 1 问(只问整体)
}
```

**屏显格式**(沿用既有):
```
=== code-design 设计目标确认 ===
整体设计目标:<--minimal/--extensible/--balanced>
维度优先级:
  功能性:<高/中/低>
已回写至 design/<REQ>/RESULT.md "## 设计目标" 小节
```
