## 2. 安全边界

### 2.1 鉴权要求

- **不涉及鉴权**:本需求不新增鉴权,沿用 `code-fix` 既有的"无鉴权" 边界
- 复现动作执行的是用户自己启动的进程(本地命令),**不**涉及远程鉴权

### 2.2 输入校验

| 输入 | 校验策略 | 失败处理 |
| --- | --- | --- |
| `bugNum` | 匹配 `^BUG-\d{5}$` | 不匹配 → `code-fix` 步骤 1 已校验,本子节**不**重复 |
| `startCommand` | 由 `detectStartability()` 推导,**不**接受用户输入 | N/A(无用户输入) |
| `reproSteps` | 用户在步骤 1 / 5 输入(自然语言);本子节按 `type` 字段分发 | 类型未知 → 屏显 `⚠`,跳过该步骤 |

### 2.3 敏感数据处理

- 复现产物**不**视为"敏感数据"(本仓库是开发工具,**不**上传到外部)
- 产物放 `fix/<BUG-NNN>/reproduce/`,**不**自动清理(用户后续可手动)
- 沿用 NFR-9:不修改任何项目配置 / .gitignore

### 2.4 审计日志

- 每次复现动作在 `fix/<BUG-NNN>/RESULT.md` 文档头"修复日志" 段追加 1 条(由 `code-fix` 步骤 6 末尾完成)
- `RESULT-meta.json` 记录完整运行时间 / 退出码 / 产物清单

### 2.5 依据规范

- NFR-9(不修改项目源码)
- 沿用 `code-fix` 既有的"无鉴权" 边界
- 沿用 NFR-7(产物放子目录,不入 git 仓库配置)

## 3. 性能与资源

### 3.1 关键路径预估

| 路径 | 耗时 | 说明 |
| --- | --- | --- |
| `detectStartability()` 探测 | < 500ms | 8 个 `test -f` 调用,每次 < 50ms |
| 子进程启动 | 5s(等待) | 给程序启动时间 |
| 单步骤执行 | < 30s(推荐) | 在 60s 总超时内 |
| 复现总耗时 | < 60s | 含启动 + 所有步骤 |
| 日志合并 | O(N)(N=日志行数) | 内存处理,无需 IO 密集 |
| 屏显契约 | O(1) 行 | 3 行屏显,每行 < 80 字符 |

### 3.2 资源限制

| 资源 | 限制 | 触发降级 |
| --- | --- | --- |
| 子进程 | 60s 超时 | E-3 |
| 日志大小 | 10MB(> 截断到 1MB) | E-5 |
| 子进程数 | 1(本需求不并发) | N/A |
| 磁盘空间 | 沿用 OS 限制 | 不特殊处理(产物小,< 100MB) |

### 3.3 缓存策略

- **不**缓存探测结果(每次 `code-fix` 调用都重新探测,确保结果新鲜)
- 沿用 `code-fix` 既有的"无缓存" 边界
