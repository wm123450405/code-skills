## 4. 风险与权衡

| 风险 | 等级 | 缓解 |
| --- | --- | --- |
| 措辞过严,误伤"既有 `code-unit` 引用但语义已变" | 中 | 改写时**只**改字面,核心约束保留 |
| `code-it` 接管守卫后 7 项检查与原行为不一致 | 中 | 字节级沿用 |
| `code-auto` 步骤 7 输出报告"单元测试(code-unit):N2 次"如何处理 | 中 | 字段**改写**为"单元测试(由 `code-it` 内化):N2 次" |
| 历史 `auto-report.md` 中"code-unit"日志字节级保留? | 低 | NFR-2 沿用;**不**追溯重写 |
| `code-it` 净增 ~200 行,可能突破单 SKILL.md 加载阈值 | 低 | `code-it` 当前 938 行,加 200 后 ~1138,仍在范围内 |
| 5 SKILL.md 改造,易引入回归 | 中 | 字节级保留 frontmatter / §"工作流程" 既有步骤;INV-1 ~ INV-3 严守 |
| `code-check` 评审时仍读历史 `test/<TASK-...>/RESULT.md` 是否行为变化 | 低 | FR-7 字面改写**只**改文案,不影响历史路径 |
| `code-unit` 步骤 0a 既有"7 项检查"是否覆盖本仓库元技能场景 | 低 | 本仓库是元技能仓库,无工程代码;沿用原守卫 7 项是**字节级保留**,不会误判 |
