## 关键设计问题清单
1. **模块切分**:仓库根 vs 插件子目录的两层结构 → 根 README = 门面,`plugins/code-skills/README.md` = 详细(GitHub 标准模式)
2. **状态归属**:本需求零代码,无状态机/事务边界
3. **依赖方向**:本需求零依赖,无依赖方向
4. **扩展点位置**:本需求不涉及扩展点

## 候选方案
| 候选 | 描述 | 选择 |
| --- | --- | --- |
| A. 仓库根 README = 门面 + 插件子 README = 详细 | 经典 GitHub 两层结构 | **选定** |
| B. 仅在根 README 写完所有内容 | 违反"极简"原则(NFR-2) | 否决 |
| C. 移动 CLAUDE.md + 在 `plugins/code-skills/` 保留软链 | 违反 NFR-8(不提供重定向) | 否决 |

## 选定方案与理由
- **方案 A**:符合 GitHub 惯例 + 满足 NFR-2(< 50 行)+ 满足 FR-3(用户明确要求 CLAUDE.md 在根)

## 决策引用
- 移动而非复制 CLAUDE.md:FR-3 AC-3.1(用 `git mv`)
- 中英 README 同次提交:`doc-conventions §规则 1`
- 极简 < 50 行:NFR-2
- 不提供重定向/软链:NFR-8
- 详细文档保留原样:FR-4 + NFR-4
