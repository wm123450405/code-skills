## 2. 候选方案 + 选定方案 + 否决理由

### DQ-1:快模式入口触发方式

| 候选 | 描述 | 优点 | 缺点 |
| --- | --- | --- | --- |
| A | 全局开关(环境变量 `CODE_FAST_MODE=1` 永久开启) | 简单;一次性配置;`code-auto` 可在主调前设置 | 缺乏"单次覆盖"灵活性 |
| B | 单次开关(CLI 标志 `--fast REQ-NNNNN`) | 灵活;可与默认完整模式共存 | 每次都要传;用户容易忘 |
| C | 双模式并存(新技能 `/code-design-fast`) | 物理隔离清晰 | 违反"优化既有技能"语义;增加新技能目录需 marketplace.json 同步,违反 FR-6.AC-6.2 |
| D | 自动推断(根据项目类型/规模自动决定) | 0 配置;智能 | 推断规则难定义;**不**可预测;违反"行为明确"原则 |
| **E** ✅ | **环境变量 + CLI 标志组合**(优先级 CLI > 环境变量 > 默认) | 兼顾简单 + 灵活;`code-auto` 可在主调前设置;用户可单次覆盖 | 解析逻辑略复杂 |

**选定**:E(环境变量 + CLI 标志组合)

**依据规范**:
- `skill-conventions §规则 1`:SKILL.md frontmatter 不变(本设计**不**改 frontmatter,仅追加"步骤 0.5 模式选择"段)
- FR-1.AC-1.1 ~ AC-1.6(本设计锁定)
- FR-6.AC-6.1(0 新增技能)
