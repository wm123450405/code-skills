### 讨论项 D-5:快模式与 `code-auto` 的子技能调用接口

- **问题**:`code-auto` 调子技能时如何传递"快模式"信号?
- **选项**:
  - **A. 通过环境变量**:`code-auto` 在主调前设置 `CODE_FAST_MODE=1`,子技能自动继承
  - **B. 通过 CLI 标志**:`code-auto` 调 `/code-design --fast REQ-NNNNN`
  - **C. 通过子技能传参**:`code-auto` 调 `/code-design --mode=fast REQ-NNNNN`
- **决定**:**A(通过环境变量)**
- **理由**:
  1. 环境变量是**最简单**的跨进程传参方式(`code-auto` 主进程 + 子技能进程共享环境变量)
  2. NFR-4 锁定:`code-auto` **不**自动设置 `CODE_FAST_MODE=1`;**用户**在主调前显式设置
  3. CLI 标志方案(B)的缺点:`code-auto` 调子技能时 CLI 标志需要 Skill 工具支持(本仓库 12 个 `code-*` 既有实践**不**依赖 CLI 标志透传)
  4. 子技能传参方案(C)的缺点:需要修改 `code-auto` 调子技能的传参逻辑,违反 FR-6.AC-6.1(0 修改子技能)
- **影响**:`code-auto` 用户用法:`export CODE_FAST_MODE=1 && /code-auto "<需求内容>"`
