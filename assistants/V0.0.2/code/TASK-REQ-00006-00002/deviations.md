## 偏离 1(微小细化,非违反):第 7 节"首次进入软件系统"的访问 URL placeholder 拆分

- **类别**:实现细节细化(不偏离设计,只是 placeholder 粒度优化)
- **依据**:
  - 设计要求:data-changes.md §4.1 章节 7 列出 placeholder `<首次访问 URL>`(单一占位)
  - 规范要求:无
- **实际做法**:模板中写成 `http://<server>:<port>`(拆为 server 与 port 两个 placeholder)
- **偏离理由**:
  - 用户补全时通常**分别知道**域名/地址与端口(可能来自不同配置源)
  - 拆为两个 placeholder 让"补全"操作更可执行
  - 模板中仍显式提示 `例如 http://192.168.1.100:8080`,用户可一目了然
- **影响**:
  - `code-publish` 替换 placeholder 的逻辑**不依赖**此粒度(它只替换 `<本版本号>`,其他保留)
  - 用户补全体验更佳
  - **不影响任何 FR / AC / NFR**
- **授权**:无需用户授权(纯细化,在设计"placeholder 集合"允许的范围内;`code-publish` 行为不变)
- **时间**:2026-06-04 17:34

- **影响**:

  - `code-publish` 复制模板时**不区分**附录与正式章节
  - **不影响任何 FR / AC / NFR**
- **授权**:无需用户授权(纯增量,可选)
- **时间**:2026-06-04 17:34

## 偏离 3(细化,非违反):第 6 节"启动运行"提供 3 种"其他启动方式"

- **类别**:实现细节细化(不偏离设计,符合 NFR-5 通用性)
- **依据**:
  - 设计要求:data-changes.md §4.1 章节 6 仅给 1 个默认示例(systemd)
  - 规范要求:无
- **实际做法**:在 systemd 默认示例**之外**,以"其他启动方式(若使用)"列出 3 种替代方案(Docker / nohup / Docker Compose)
- **偏离理由**:
  - 不同软件形态的启动方式差异巨大;只给 systemd = 强制用户改模板
  - 列 3 种常见替代方案 = 用户大概率能找到"自己用的那一种"或"可借鉴的命令"
  - **不**违反 AC-3.2(systemd 是默认示例;其他是补充)
- **影响**:
  - 模板更长(~245 行 vs needs §6.1 的 ~70 行;但仍 < 300 行,可控)
  - **不影响任何 FR / AC / NFR**
- **授权**:无需用户授权(纯细化,在"最常见部署方式默认内容"原则内)
- **时间**:2026-06-04 17:34
