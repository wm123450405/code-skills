3. **可读性**:5 列(读 / 写 / 下游)提供更多上下文(用户能快速看出本技能是"只读"+"屏幕输出"+"引导其他技能")
4. **PLAN.md 字面 vs 实际可行的冲突**:在 `code-it` 阶段,"机械照 PLAN.md 字面"会破坏"结构对仗"这一更高层约束(由 `doc-conventions §规则 1` 锁定);需"超越 PLAN.md 字面"以满足上层约束
