## 偏离 1(增量,非违反):Q&A.md 模板提供"完整生成结果示例"

- **类别**:实现细节增量(不偏离设计,符合 NFR-9 可读性优先)
- **依据**:
  - 设计要求:data-changes.md §4.3 只列"文首 + 引言 + 占位章节";**未要求**完整生成结果示例
  - 规范要求:无
- **实际做法**:在模板的"占位章节"内,提供"完整生成结果示例"(假设 qanda/ 有 2 个文件)作为 fenced code block
- **偏离理由**:
  - 模板用户(运维 / 现场支持)通常**不知道** `code-publish` 会如何把 qanda/ 渲染成 Q&A.md
  - 给个"示例"让用户**先看到期望结果**,再决定"是否值得向 qanda/ 添加内容"
  - 示例用 fenced code block,**不**会被错误地当成实际 Q&A 内容
- **影响**:
  - 模板从 ~30 行 → ~63 行(略增)
  - **不影响任何 FR / AC / NFR**
  - 用户可整段删除示例(无依赖)
- **授权**:无需用户授权(增量)
- **时间**:2026-06-04 17:52

## 偏离 2(增量,非违反):占位章节内提供"4 步添加 Q&A 内容"指南

  - 规范要求:无

- **影响**:

- **授权**:无需用户授权(增量)
- **时间**:2026-06-04 17:52
