# 接口详细规格 — REQ-00017
版本:V0.0.2

## 接口:P-1 推进看板(本需求新增,SKILL.md 内部步骤)

- **形式**:SKILL.md 内部步骤(非对外 API)
- **触发位置**:`/code-it` 末尾兜底"步骤 5 commit 成功"后自动执行
- **签名**:`P-1 推进看板(<任务编码>) -> void`
- **入参**:
  - `<任务编码>`:字符串,本任务的编码(如 `TASK-REQ-00017-00001`)
- **出参**:无返回值;副作用 = 写回看板文件
- **副作用**:
  - 读:`./assistants/<版本号>/RESULT.md`
  - 写:`./assistants/<版本号>/RESULT.md`(原子的 read-modify-write)
- **错误码**:无对外错误码;失败 → stderr 告警 + 跳过
- **失败处理**:
  - 解析失败 / 写回失败 → stderr 告警,**不重试**,**不阻断**
  - 看板本任务行缺失 → stderr 告警,跳过
  - 看板本任务行"开发状态"已是"已完成" → 幂等跳过
  - 看板本任务行"开发状态"非"待开始"也非"进行中" → stderr 告警,跳过
- **幂等性**:是(同一任务被多次执行,P-1 行为幂等)
- **串行约束**:必须 `/code-it` 末尾兜底 commit **成功**后才执行(commit 失败 → 立即退出,不执行 P-1)
- **依据规范**:`dashboard-conventions.md §规则 1` / `encoding-conventions.md §规则 1+3`

## 看板写回的副作用结构

```yaml
看板"任务清单"区段:
  - 定位: ^## 任务清单$
  - 操作: 找到本任务行 + 修改"开发状态"列 + 写入"完成时间"列
  - 原子性: 整文件 read-modify-write

看板"变更记录"区段:
  - 定位: ^## 变更记录$
  - 操作: 追加一行"任务完成"事件
  - 格式: "YYYY-MM-DD HH:mm  任务完成  <任务编码> <任务标题>(开发状态:已完成)  <任务编码>"
```
