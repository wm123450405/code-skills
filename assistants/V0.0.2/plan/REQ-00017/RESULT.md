**新增内容**(伪代码):
```markdown
#### 步骤 P-1 推进看板(REQ-00017 新增,2026-06-05 起生效)

**触发条件**:步骤 5 `git commit` **成功**(返回 0)后,自动执行

**职责**:把本任务的看板"开发状态"列从"待开发"推进到"已完成";不动"测试状态"列(留给 `/code-unit`)

**算法**(伪代码):
```
P-1 推进看板(<任务编码>):
  1. 解析 <版本号> = Read("./assistants/.current-version").trim()
  2. dashboard_path = "./assistants/" + <版本号> + "/RESULT.md"
  3. content = Read(dashboard_path)
  4. anchor = locate_section(content, "^## 任务清单$")       # 沿用 dashboard-conventions §规则 1
  5. if anchor is None:
       stderr("⚠ 看板未找到'任务清单'区段,跳过推进")
       return
  6. rows = parse_table_rows(anchor)                          # 表格行 ^\| .* \|$
  7. target_row = None
  8. for row in rows:
       task_id = row.cells[0]                                  # 任务编码列
       if matches(task_id, <任务编码>, format=TASK-...|旧格式):
         target_row = row
         break
  9. if target_row is None:
       stderr("⚠ 看板未找到本任务行 " + <任务编码> + ",跳过推进")
       return
 10. dev_status_col = target_row.cells[<开发状态 列索引>]      # 详 §3.4 列定义
 11. if dev_status_col == "已完成":
       # 幂等:不重复写
       return
 12. if dev_status_col != "待开始" and dev_status_col != "进行中":
       stderr("⚠ 看板本任务开发状态为 '" + dev_status_col + "',非预期值,跳过推进")
       return
 13. target_row.cells[<开发状态 列索引>] = "已完成"
 14. target_row.cells[<完成时间 列索引>] = now("YYYY-MM-DD HH:mm")
 15. write_back(dashboard_path, content)                       # 原子操作
 16. append_change_log(dashboard_path, "任务完成", <任务编码> + " <任务标题>(开发状态:已完成)", <任务编码>)
```

**失败处理**(对应 D-5 / E-2):
- 解析失败 / 写回失败 / 权限不足 → stderr 告警 `⚠ 看板推进失败(<原因>),请人工核对`,**不重试**,**不阻断**;`/code-it` 整体仍视为完成

**幂等性**(对应 D-6 / E-4):
- 同一任务被多次执行(修正 bug / 重跑)时,本任务"开发状态"已是"已完成"则**跳过**(不重复写)

**边界**(对应 D-2 / D-3 / E-3 / E-5):
- E-3 看板"任务清单"区段**缺失**:跳过 + stderr 告警
- E-5 看板中**找不到**本任务行:跳过 + stderr 告警
- D-3 不动"测试状态"列(由 `/code-unit` 推进)

**与既有步骤的关系**:
- 既有"步骤 5 执行 commit 成功"路径**不变**
- P-1 步骤**只**在 commit 成功后执行(D-8 串行)
- P-1 步骤**不向用户提问**(code-auto 自动模式必需)

## 4. 算法与逻辑

### 4.1 P-1 推进看板的关键算法

详 §3.3 伪代码。关键点:
- **解析锚点**:`^## 任务清单$`(沿用 `dashboard-conventions §规则 1`)
- **任务编码匹配**:双格式正则 `^TASK-(REQ|BUG)-\d{5}-\d{5}$` + `^(REQ|BUG)-\d{5}-\d{5}$`
- **列定位**:从模板 `version-RESULT.md` 中获取"开发状态"列的固定列索引(本需求**不**改模板,列索引保持)
- **原子写回**:整文件 read-modify-write;中途失败时下一次执行可重试(幂等)

### 4.2 状态机:看板"任务清单"区段本任务行

```
[/code-plan 步骤 10A 拆任务]
  ↓
[/code-plan 步骤 16A 同步看板]
  看板行:开发状态=待开始, 测试状态=未编写/不适用, 完成时间=空, 触发/来源=详细设计
  ↓
[/code-it 末尾兜底 commit 成功]
  ↓
[/code-it P-1 推进看板]    ← 本需求
  看板行:开发状态=已完成, 测试状态=未编写/不适用(不变), 完成时间=now(), 触发/来源=详细设计(不变)
  ↓
[/code-unit 完成(若带测试)]    ← 本需求**不**改
  看板行:开发状态=已完成(不变), 测试状态=已运行-通过/不适用, 完成时间=不变
```
