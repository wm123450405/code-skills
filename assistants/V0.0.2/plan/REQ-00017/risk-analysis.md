# 风险分析 — REQ-00017
版本:V0.0.2

## 异常处理

| 异常路径 | 描述 | 处理策略 | 监控指标 |
| --- | --- | --- | --- |
| `/code-it` 末尾兜底 commit 失败 | 步骤 5 返回非 0 | P-1 步骤不执行;`/code-it` 整体失败 | 由 commit 失败决定 |
| P-1 解析看板失败 | Read 失败 / 正则匹配失败 | stderr 告警 + 跳过 | stderr 关键字 `⚠ 看板推进失败` |
| P-1 写回看板失败 | Write 失败 / 权限不足 | stderr 告警 + 跳过 | stderr 关键字 `⚠ 看板推进失败` |
| P-1 看板本任务行缺失 | 解析后未找到匹配行 | stderr 告警 + 跳过 | stderr 关键字 `⚠ 看板未找到本任务行` |
| P-1 看板本任务行"开发状态"已是"已完成" | 二次执行 / 重复推进 | 幂等跳过(不报错) | 无日志 |
| P-1 看板本任务行"开发状态"非"待开始"也非"进行中" | 异常值 | stderr 告警 + 跳过 | stderr 关键字 `⚠ 非预期值` |
| 看板"任务清单"区段缺失 | 看板被外部破坏 | stderr 告警 + 跳过 | stderr 关键字 `⚠ 看板未找到` |

## 安全边界

- **写回操作**:仅写回 `./assistants/<版本号>/RESULT.md`,不写其他文件
- **权限**:不引入新权限要求
- **输入校验**:任务编码正则严格匹配 `^TASK-(REQ|BUG)-\d{5}-\d{5}$` 或 `^(REQ|BUG)-\d{5}-\d{5}$`,不匹配则跳过
- **敏感数据**:无
- **审计日志**:看板"变更记录"区段追加"任务完成"事件(沿用既有实践)

## 性能与资源

- **P-1 步骤耗时**:单文件 read-modify-write,< 1ms(典型 V0.0.2 看板 ~50KB)
- **网络请求**:0
- **新增依赖**:0
- **CPU 占用**:0
- **磁盘 IO**:2 次(读 + 写)

## 回退策略

| 触发条件 | 步骤 | 验证 |
| --- | --- | --- |
| 2 个 SKILL.md 增量追加导致 `/code-plan` / `/code-it` 行为异常 | `git revert <commit-hash>` 即可 | Read SKILL.md 确认回到增量追加前 |
| P-1 步骤被误触发(commit 失败时) | 既有 D-8 串行约束保证不触发 | 跑一次失败 commit,验证 P-1 不执行 |
| P-1 步骤推进错误任务 | 幂等 + stderr 告警,人工可重跑修正 | 跑一次完整需求,人工核对看板"任务清单"区段 |
