## 异常处理要点
- **git 命令失败**:退出码非 0 → 透传 stderr + 退出码非 0
- **pre-commit hook 失败**:不重试,提示用户手动
- **LLM 无法解决冲突**:留 unmerged + 继续处理其他文件(不阻塞)
- **二进制文件冲突**:留 unmerged + 提示用户手动(不自动)
- **看板自检不一致**:打印报告 + 不修复(留 `code-rule` 处理)
- **Ctrl+C**:立即停止,已 commit 不回滚(commit 是 git 原子的)

## 安全要求
- **输入校验**:参数数量(0/1/2+)+ worktree 路径 + 主干分支存在性
- **敏感数据**:不打印任何 token / password(本技能不涉及)
- **审计日志**:所有 git commit 都带 author / timestamp(由 git 自身保证)

## 性能与资源
- **关键路径**:git fetch + merge + 5 区段扫描
- **性能预估**:
  - git fetch:网络依赖(秒级 ~ 分钟级)
  - git merge:文件数依赖(秒级 ~ 十秒级)
  - 看板自检(5 区段 × 表格行计数):< 100 ms
  - LLM 智能合并:文件数 + 复杂度依赖(秒级 ~ 分钟级)
- **资源限制**:0 内存压力(纯 IO + LLM 推理)

## 回退策略
- **git commit 失败**:不重试,提示用户手动 `git commit` 或 `git reset`
- **git merge 失败**:不自动回滚(用户手动 `git merge --abort` 或解决冲突)
- **FR-4 冲突无法 LLM 解决**:留 unmerged,用户手动 `Edit` + `git add`
- **FR-7 main 合并失败**:不自动回滚(用户手动 `git merge --abort`)
- **本技能不主动回滚任何 git 状态**(避免数据丢失)
