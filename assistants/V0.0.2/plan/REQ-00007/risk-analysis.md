# 风险分析 — REQ-00007

更新时间:2026-06-05 10:15
版本:V0.0.2

## 1. 异常处理(E-1 ~ E-13,沿用需求 §9)

### E-1:无 `.current-version`
- **触发条件**:`Read "./assistants/.current-version"` 失败
- **检测手段**:Claude Code `Read` 工具返回错误
- **处理策略**:提示用户调 `code-version`,退出码 3
- **监控指标**:本任务的 `code/.../RESULT.md §失败` 段
- **对应任务**:T-001(在 SKILL.md §7 步骤 0 中体现)

### E-2:`git pull` 冲突
- **触发条件**:步骤 0a `Bash git pull` 退出码 ≠ 0 + stderr 含 "CONFLICT"
- **处理策略**:打印冲突文件列表 + 提示"请手动解决冲突后重跑"+ 退出码 2
- **依据**:沿用 REQ-00005 错误处理范式
- **对应任务**:T-001(SKILL.md §7 步骤 0a)

### E-3:`git pull` 网络失败
- **触发条件**:stderr 含 "Could not resolve host" / "Connection" / "timeout"
- **处理策略**:打印"网络/remote 不可达"+ 退出码 2
- **对应任务**:T-001

### E-4:`git pull` 凭据失败
- **触发条件**:stderr 含 "Permission denied" / "Authentication failed" / "403"
- **处理策略**:打印"权限/凭据问题"+ 退出码 2
- **对应任务**:T-001

### E-5:子技能退出码 ≠ 0
- **触发条件**:`Skill` 工具返回的子技能退出码 ≠ 0
- **处理策略**:立即中断 + 报告(不重试,Q-2 锁定 A)+ 退出码 1
- **监控指标**:`code-auto` 屏幕报告 + 看板"变更记录"区段
- **对应任务**:T-001(SKILL.md §7 步骤 7 异常路径)

### E-6:用户 `Ctrl+C` (SIGINT)
- **触发条件**:用户在 `code-auto` 跑长时按 Ctrl+C
- **处理策略**:打印"⏹ code-auto 用户中止"报告 + 退出码 130
- **不写** `auto-report.md`(NFR-7 强约束)
- **对应任务**:T-001(SKILL.md §7 步骤 7 中止路径)

### E-7:评审循环无收敛
- **触发条件**:`code-review` 多次轮转,每轮都产生"必须改"任务
- **处理策略**:**持续循环**(Q-1 锁定 A,无轮数上限)+ 接受"修改引入新问题"风险
- **用户中止**:Ctrl+C(E-6)
- **对应任务**:T-001(SKILL.md §5 状态机评审循环分支)

### E-8:子技能耗时过长
- **触发条件**:任一子技能挂起 / 长时间无响应
- **处理策略**:**接受**(Q-3 锁定 A);中止靠 Ctrl+C(E-6)
- **对应任务**:T-001

### E-9:`code-auto` 自身崩溃
- **触发条件**:Claude Code 模型层异常 / 网络中断 / token 上限
- **处理策略**:打印"部分报告"(若有)+ **不写** `auto-report.md`
- **对应任务**:T-001

### E-10:`auto-report.md` 写入失败
- **触发条件**:`Write` 工具失败(权限/磁盘满)
- **处理策略**:**警告不中断**(D-6 选定 A);屏幕报告已落,核心需求已满足
- **对应任务**:T-001

### E-11:`plan/PLAN.md` 缺失
- **触发条件**:步骤 4 前置读 `plan/REQ-NNNNN/PLAN.md` 失败
- **处理策略**:中断 + 报告 + 退出码 1
- **对应任务**:T-001

### E-12:`review/REVIEW-REPORT.md` 缺失
- **触发条件**:步骤 5/6 循环读 `REVIEW-REPORT.md` 失败
- **处理策略**:中断 + 报告 + 退出码 1
- **对应任务**:T-001

### E-13:缺参数
- **触发条件**:`/code-auto` 无参数调用
- **处理策略**:提示用法示例 + 退出码 4
- **对应任务**:T-001

## 2. 安全边界

### 2.1 鉴权
- **N/A**:`code-auto` 由 Claude Code 模型层触发,无显式鉴权
- **隐式约束**:`code-auto` 写入 `auto-report.md` 时,沿用 CWD 下的文件系统权限(由 Claude Code 进程用户决定)

### 2.2 授权
- **N/A**:`code-auto` 不做细粒度授权检查
- **隐式约束**:`code-auto` 不修改 `git config` / 不写 `.env` / 不读密钥

### 2.3 输入校验
- **`<需求内容>`**:自然语言,无长度上限(由 Claude Code 模型层管理)
- **`<版本号>`**:从 `.current-version` 读,1 行字符串
- **`<需求编码>` / `<任务编码>`**:沿用 `encoding-conventions §规则 1-4` 5 位纯数字

### 2.4 敏感数据处理
- **不读**任何 `secrets` / `.env` / `id_rsa` / `token` 文件
- **不写**任何敏感数据到 `auto-report.md`(报告内容仅含技术元信息)
- **不打印**敏感数据到屏幕(进度输出仅含步骤名 + 参数,无 token)

### 2.5 防注入
- **N/A**:本仓库不涉及 SQL / NoSQL / 反序列化
- **命令注入防护**:`Bash` 工具调用 `git pull` 时,沿用 REQ-00005 错误处理范式(无参数污染)

### 2.6 审计
- **进度日志**:每步一行,含子技能名 + 参数(用户可全程跟看)
- **完成报告**:`auto-report.md` 留痕
- **git 历史**:由 `code-it` 自身 commit(沿用各子技能约定)
- **看板**:`assistants/<版本号>/RESULT.md` 任务清单 + 变更记录

### 2.7 依据规范
- `marketplace-protocol.md §规则 1`(不引入未知字段)
- `encoding-conventions.md §规则 1-4`(输入校验锚点)

## 3. 性能与资源

### 3.1 关键路径耗时目标
- **`code-auto` 整体执行**:无具体上限(由子技能耗时决定)
- **典型场景**:1 需求 + 5 任务 + 1 轮评审 = 约 30-60 分钟(参考 `code-require` / `code-design` / `code-plan` / `code-it` / `code-unit` / `code-review` 各自耗时)
- **大需求场景**:10+ 任务 + 多轮评审 = 数小时

### 3.2 并发上限
- **N/A**:NFR-2 强约束串行
- **单实例**:`code-auto` 1 次执行 = 1 个 Claude Code 会话

### 3.3 资源限制
- **内存**:Claude Code 上下文 token 上限(由 Claude Code 模型层管理)
- **磁盘**:`auto-report.md` 2-5 KB,可忽略
- **网络**:`git pull` × 1(步骤 0a)+ 子技能内部 `git pull` × N(沿用 REQ-00005)

### 3.4 缓存策略
- **N/A**:`code-auto` 不维护内存缓存(无状态执行器,Q-11 锁定)
- **不写运行日志 / 调试日志**

### 3.5 批量/异步
- **不批量**:串行调子技能(Q-7 锁定 A)
- **不异步**:所有步骤同步等待子技能返回

### 3.6 降级策略
- **N/A**:`code-auto` 不存在"降级"概念 — 异常即中断(Q-2 锁定 A)
- **可中止**:用户 Ctrl+C 即可中止

## 4. 回退策略

### 4.1 触发条件
- `code-auto` 实施后发现重大问题(性能 / 安全 / 行为错误)
- 6 个子技能中的某个被破坏(本设计无此问题,因零修改)
- `marketplace.json` 加载失败

### 4.2 回退步骤
**3 步**:
1. `git revert <提交哈希>`(整体回滚)
2. 或手动删除 3 处:
   - `rm plugins/code-skills/skills/code-auto/SKILL.md`
   - `Edit marketplace.json` 移除 `./skills/code-auto` 行
   - `Edit README.md` + `README.en.md` 移除 `code-auto` 行
3. **可选**:`Edit V0.0.2/RESULT.md` 移除本需求相关行(纯文档,可保留)

### 4.3 验证
- 重跑 `code-publish`(若已配前置检查应通过)
- 重启 Claude Code,确认 `code-auto` 不再可触发
- 检查 `git status` 干净

### 4.4 副作用
- **无副作用**:6 个子技能 SKILL.md 字节级保留
- **用户可重跑**:`code-auto` 不持久化任何状态
