## 5. 提交信息数据结构

### 5.1 commit message 完整 schema(各技能)

#### 5.1.1 `code-require` 任务
```yaml
type: "chore"
scope: "code-require"
subject: "<需求编码> <需求标题>"  # 例:"REQ-00005 优化 3 个技能,首步拉取+末步兜底提交"
body: |
  需求分析完成:<N> FR / <M> NFR / <K> AC
  过程文件 3 + 结果文件 1 = <X> 文件待提交
```

#### 5.1.2 `code-design` 任务
```yaml
type: "chore"
scope: "code-design"
subject: "<需求编码> <设计标题>"
body: |
  概要设计完成:<D> 项决策,<I> 条不变量
  过程文件 N + 结果文件 1
```

#### 5.1.3 `code-plan` 任务
```yaml
type: "chore"
scope: "code-plan"
subject: "<需求编码> <计划标题>"
body: |
  详细设计与编码计划完成:<T> 个任务
  过程文件 N + 结果文件 2(PLAN.md + RESULT.md)
```

### 5.2 验证字段
- `type`:`chore`(本仓库 V0.0.1 实践)
- `scope`:必须与技能目录名一致
- `subject`:必须含 `REQ-NNNNN`(5 位纯数字,严格遵循 `encoding-conventions.md §规则 1`)
- `body`:每行 ≤ 72 字符(可读性)
