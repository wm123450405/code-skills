更新时间:2026-06-05 17:20

## 接口 I-1:`code-unit` 入口(版本感知)

### 形式
Claude Code Skill 工具调用

### 签名
```
code-unit <任务编码>
```

### 入参
| 字段 | 类型 | 必填 | 约束 | 示例 |
| --- | --- | --- | --- | --- |
| 任务编码 | string | ✓ | 格式 `^TASK-REQ-\d{5}-\d{5}$` 或 `^TASK-BUG-\d{5}-\d{5}$` | `TASK-REQ-00009-00001` |

### 出参
| 形式 | 内容 |
| --- | --- |
| 屏幕输出 | (详见 `plan/REQ-00009/RESULT.md §5.2` 屏幕输出契约) |
| 文件 | `test/<任务编码>/RESULT.md`(守卫通过时)或**无文件**(守卫不通过时) |
| 看板 | `assistants/V0.0.2/RESULT.md §任务清单` 1 行更新(守卫不通过时) |
| 退出码 | 0(守卫通过 + 测试通过 / 守卫不通过)/ ≠ 0(其他) |

### 错误码 / 退出码
| 场景 | 退出码 | 含义 |
| --- | --- | --- |
| 守卫不通过 + 跳过 | 0 | 正常(FR-2.AC-2.2) |
| 守卫通过 + 测试通过 | 0 | 正常(原行为) |
| 守卫通过 + 测试失败 | ≠ 0 | 失败(原行为) |
| 守卫通过 + 崩溃 | ≠ 0 | 失败(原行为) |
| 任务编码不存在 | ≠ 0 | 错误(原行为) |
| 无 `.current-version` | ≠ 0 | 错误(原行为) |

**输入**:`code-unit TASK-REQ-00009-00001`
**前置条件**:项目根无 `package.json` / `pyproject.toml` / `Cargo.toml` / `go.mod` / `pom.xml` / `build.gradle` / `test/` 目录
**输出**:

**退出码**:0
**副作用**:`assistants/V0.0.2/RESULT.md` 任务清单 T-001 行"测试状态"列更新为 `不适用`

**输出**:

  - pyproject.toml:✗
  - Cargo.toml:✗
  - go.mod:✗
  - pom.xml:✗
  - build.gradle:✗
  - test/ 目录:✗

**退出码**:0(测试通过) / ≠ 0(测试失败)
**副作用**:`test/TASK-REQ-00009-00001/RESULT.md` 新建 + 看板"测试状态"列更新

### 版本策略
- 本接口签名沿用 V0.0.1 既有,**0** 变更
- 行为变化是**新增守卫**(从"无守卫"到"有守卫"),向后兼容(守卫通过时行为完全不变)

### 兼容策略
- **守卫不通过** = 新场景,原 `code-unit` 没有这个分支 → 兼容(`code-auto` 据此继续)
- **守卫通过** = 原场景,行为完全不变 → 100% 兼容

### 依据规范
- `skill-conventions §规则 1`(Skill 入口签名)
- `code-auto/SKILL.md §FR-6`(Q-4 锁定 A 协同约束:`code-auto` 调 `code-unit` 时,`code-unit` 退出码 = 0 让 `code-auto` 继续)
- `code-publish/SKILL.md §FR-1`(前置检查:任务"测试∈{已运行-通过, 不适用}" → 任务解决)

## 接口 I-2:看板"测试状态"列写入路径

### 形式
`code-unit` 步骤 14 既有 `Edit` 工具调用

### 路径
- `assistants/V0.0.2/RESULT.md`(L160+ "任务清单"区段)

### 调用时机
- 守卫通过 + 测试通过 → 写入"已运行-通过"
- 守卫通过 + 测试失败 → 写入"已运行-失败"
- 守卫不通过 → 写入"不适用"

### 写入列
- "测试状态"列(既有列,非新增)

### 写入值
- `已运行-通过` / `已运行-失败` / `不适用`(均为 V0.0.1 既有枚举,**0** 新增)

### 依据规范
- `dashboard-conventions §规则 1`(字段约定扩展需 3 处同步)— **不触发**(沿用既有枚举)
- V0.0.1 RESULT.md 模板既有"任务清单"区段
