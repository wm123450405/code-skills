# 设计笔记 — REQ-00014

更新时间:2026-06-05 13:15
版本:V0.0.2
需求:REQ-00014(优化技能 `/code-plan` 的任务拆分维度)

## 概要

在概要设计的基础上,本详细设计把"系统长什么样"细化为"系统怎么写"。核心决策:
- **2 个任务全部为纯文档型**(写 `code-plan` SKILL.md §10A + 8 项自检 + 收尾),0 个代码模块,0 个新依赖,0 个被修改模块(FR-4 强约束)
- **0 架构任务**(单一功能点 + 低扩展性 + 无三方组件)— 完全符合"按用户可见能力"原则
- **首次应用 REQ-00014 新规则**:本 PLAN.md 是 V0.0.2 第一个"按功能点拆分"的 PLAN

## 关键设计决策(沿用概要设计 D-1 ~ D-5 + 本设计关键观察)

### 决策 1:本需求**首次**应用"按功能点拆分"新规则

- **决策**:本 PLAN.md 是 V0.0.2 第一个应用"按功能点拆分"新规则的 PLAN
- **理由**:
  1. REQ-00014 require 阶段 Q-A3 锁定 A:仅未来需求应用新规则
  2. 之前 4 个 V0.0.2 已完成 PLAN 是按"分层拆分"原则(沿用,不追溯)
  3. REQ-00008+ 将按新规则拆分(本 PLAN 是首个"新规则样板")
- **影响**:**0**(本需求是首次应用,与既有 PLANS 无关)
- **预期效果**:T-001 "写 `code-plan` SKILL.md §10A" 应是"1 个完整一套"(展示 + 逻辑 + 说明)— 与 REQ-00007 自身的 PLAN.md 风格一致

### 决策 2:不插入架构任务

- **决策**:**不**插入架构任务
- **理由**(Q-A2 锁定 A 触发条件的 3 个检查):
  1. **触发 1**(扩展性/可维护性/可配置性 + 复杂度):本需求是 1 个简单的文档修改任务,无"较高"扩展性/可维护性/可配置性要求 — **不**满足
  2. **触发 2**(未来不同实现方式):本需求不涉及"未来切换实现方式" — **不**满足
  3. **触发 3**(对接不同三方组件):本需求不涉及三方组件(纯 Markdown 文档) — **不**满足
  - **3 个触发条件 0 满足 → 不插入架构任务**(简单需求)
- **影响**:**0**(2 任务 vs 1 任务,粒度仍可接受)
- **依据**:`require/Q-A2` 锁定 A + `design/D-2` 锁定

### 决策 3:T-001 任务边界 = "完整一套"

- **决策**:T-001 任务 = 1 个完整的功能点 = "修改 `code-plan` SKILL.md §10A"(展示 = 修改后 §10A 全文;逻辑 = 算法 1 + 10A.0-10A.3 调用表;说明 = 在 `code-plan` SKILL.md 内的"工作流步骤"参考链)
- **理由**:
  1. FR-1 强约束:**同一个功能点都应该包含一套完整的展示效果、功能逻辑、使用说明**
  2. T-001 **不**应拆为"写展示" + "写逻辑" + "写说明" 3 个子任务(那又是"分层拆分")
  3. T-001 **不**应拆为"改 §10A" + "改 frontmatter" 2 个子任务(frontmatter **不**改)
- **影响**:**0**;T-001 单任务预计 ~0.5d(改 5 行 + 写 ~25 行 = 30 行 SKILL.md 修改)

### 决策 4:T-005 任务边界 = "纯文档型自检 + 收尾"

- **决策**:T-005 任务 = 8 项不变量自检 + 偏差日志 + 收尾
- **理由**:
  1. 沿用 V0.0.2 REQ-00007 T-005 实践(纯文档型)
  2. T-005 是"整体收尾"任务,**不**与 T-001 重复
  3. 测试状态 = `不适用`(纯文档,无传统单测)
- **影响**:**0**

### 决策 5:任务编号沿用既有

- **决策**:`TASK-REQ-00014-00001` + `TASK-REQ-00014-00005`(留 00002-00004 给未来可能的任务追加)
- **理由**:
  1. `encoding-conventions §规则 1`:`TASK-(REQ|BUG)-NNNNN-NNNNN` 格式
  2. `code-plan` SKILL.md §10A(修改后)任务编号自 00001 起递增,五位补零
  3. T-005 用 `00005` 而非 `00002` — 显式"本任务预留空位"语义(未来追加任务可填 00002-00004)
- **影响**:**0**;编号连续性 = T-001 / T-005(中间 3 个空位)
