## §9 安全要求

### 鉴权
- **无 API 鉴权**(本技能是 Claude Code 技能,鉴权由 Claude Code 进程承载)
- **谁能调用**:本机 Claude Code 进程内的任何用户(无 user 概念)

### 授权
- **谁能做什么**:任何人可调用;但**只能读**(NFR-6/7 严守)
- **副作用**:无(NFR-7 幂等)

### 输入校验
- **参数**:严格 `^REQ-\d{5}$`(FR-6.AC-6.3)
- **文件路径**:由 `.current-version` 决定,**不接受**用户传入路径(避免路径注入)

### 敏感数据处理
- **本技能不读任何敏感数据**:看板 / 模板 / 需求文档均非敏感
- **日志脱敏**:**不写日志**(NFR-7,屏幕输出即日志)

### 防注入
- **不执行任何命令**(`Bash` 工具**不使用**)
- **不写任何文件**(`Write` / `Edit` 工具**不使用**)
- **不调用网络**(`WebFetch` / `WebSearch` / `Bash` 工具**不使用**)

### 审计
- **不记录审计日志**(NFR-7 幂等;若未来需要,由 `code-review` 等其他技能承担)

### 依据规范
- NFR-6:不动 `marketplace.json` / `plugin.json` / 其他 10 个 SKILL.md
- NFR-7:幂等;多次执行结果完全相同
- `marketplace-protocol.md §规则 1`:本技能不触发(走 Claude Code 技能自动发现协议)

## §10 性能与资源(NFR-4 < 5 秒)

### 关键路径耗时目标
| 指标 | 目标 | 实测(预估) | 余量 |
| --- | --- | --- | --- |
| P50(空版本,只读 .current-version) | < 1s | < 200ms | 5x |
| P50(总览模式,V0.0.2 规模) | < 1s | < 500ms | 2x |
| P95(总览模式,典型 V0.0.x 规模,< 50 需求) | < 3s | < 2s | 1.5x |
| P99(总览模式,典型 V0.0.x 规模) | < 5s | < 3s | 1.7x |
| P95(需求模式,典型 V0.0.x 规模) | < 3s | < 1s | 3x |
| 硬性上限 | < 5s | < 3s | 1.7x |

### 并发上限
- **同时处理**:1 个(本技能无并发原语,串行执行)
- **QPS**:不适用(用户手动触发,无高并发)

### 资源限制
- **单实例内存**:典型 < 1 MB(单次 1~2 个文件 Read + 内存解析)
- **CPU**:不敏感(纯字符串操作)
- **IO**:每次调用 1~3 次 `Read`(总览:1 次;需求:3 次并行)

### 缓存策略
- **无缓存**(NFR-7 幂等,缓存会破坏"读最新看板"语义)

### 批量/异步
- **需求模式 3 次 `Read`**:在同一消息块内并发触发(Claude Code 工具支持)

### 降级策略
- **超载时**:不适用(本技能无 QPS 压力)
- **看板超大时**(> 10000 行,极罕见):线性扫描可能慢;但 V0.0.x 典型 < 500 行,**不需**降级

### 依据规范
- NFR-4:< 5 秒硬性上限
- NFR-7:幂等;不引入缓存

## §12 回退策略

### 触发条件
- `code-review` 阶段发现本技能行为与设计不符(例如:误写文件 / 改动了其他模块 / 解析错误)
- 性能严重超出 NFR-4(> 10s)
- 安全测试发现 NFR-6 边界被破坏

### 回退步骤
- **L1 软回退**(技能文件有问题,但无副作用):
  1. `rm plugins/code-skills/skills/code-dashboard/SKILL.md`
  2. (可选)`git commit -m "chore: revert code-dashboard per review finding"`
  3. 用户后续可重新 `code-design` 修订
- **L2 硬回退**(技能改了不该改的):
  1. 同 L1 步骤 1
  2. `git checkout HEAD~1 -- marketplace.json plugin.json` 恢复
  3. `git checkout HEAD~1 -- plugins/code-skills/skills/*/SKILL.md` 恢复其他技能
  4. `git commit -m "revert: full revert of REQ-00004"`

### 验证
- 1 次 `git status` 确认 `clean`
- 手动调 1 次 `/code-version V0.0.x` + `/code-require ...` 确认既有技能未受影响

### 依据规范
- NFR-7 幂等性:本技能无状态,回退后**不影响**后续调用
