## 关联需求清单

### REQ-00007(版本:V0.0.2)— `/code-auto` 自动开发技能
- **关联点**:
  - **被高频调用的子技能**:`code-auto` 在任务循环中调 `code-it` + `code-unit`(每个任务 1-2 次)
  - **FR-4.AC-4.2**:`code-auto` 现行逻辑是"若 `code-it` 输出含'测试需要=Y'则调 `code-unit`"
  - **本需求**会让 `code-unit` 自身有"守卫"能力,`code-auto` 调用时守卫可能不通过 → 跳过
- **对本需求的影响**:
  - **`code-auto` FR-4.AC-4.2 的"测试需要=Y"判断逻辑是否被覆盖**?见 clarifications.md Q-6
  - **建议派生**:在 `code-review REQ-00009` 阶段,可派生"`code-auto` 升级:增加'任务级跳过单测'感知"任务(由用户决定)
  - **直接影响**:本需求不修改 `code-auto`,但本需求落地后 `code-unit` 自身行为变化,`code-auto` 调用体验更"智能"
- **来源**:`./assistants/V0.0.2/require/REQ-00007/RESULT.md` §FR-4 / §NFR-4

### REQ-00005(版本:V0.0.2)— 优化 3 技能,加首步拉取与末步提交
- **关联点**:
  - **首步拉取位置 = 步骤 0 之前**;本需求的"守卫"在概念上类似"步骤 0a"
  - **`code-unit` 不在 REQ-00005 改写范围**;本需求扩展 `code-unit` 是**独立**新增,与 REQ-00005 并行
- **对本需求的影响**:
  - **守卫位置约定**:本需求守卫放在"步骤 0 版本上下文检测"**之前**(与 REQ-00005 的"步骤 0a 拉取"同位)
  - **结构一致性**:本需求守卫逻辑应遵循 REQ-00005 的"增量追加,不重写"原则
  - **不触发** REQ-00005 扩展到 `code-unit`(留作 follow-up)
- **来源**:`./assistants/V0.0.2/require/REQ-00005/RESULT.md` §FR-1 / §NFR-2

### REQ-00004(版本:V0.0.2)— `/code-dashboard` 开发看板技能
- **关联点**:
  - **看板"任务清单"区段**含"测试状态"列(枚举:`未编写` / `已编写` / `已运行-通过` / `已运行-失败` / `不适用`)
  - **`code-unit` 跳过时**写入看板的"测试状态"是什么?
  - **`code-dashboard` 的统计逻辑**(NFR-4)消费这个字段
- **对本需求的影响**:
  - **NFR-3 关键决策**:"跳过时"测试状态值的选择会**直接**影响 `code-dashboard` 统计
  - **如果选 A(复用"不适用")**:`code-dashboard` 现有逻辑**无变化**(NFR-8 兼容)
  - **如果选 B(新增"已完成(跳过)")**:`code-dashboard` 现有逻辑**不识别**,需升级统计聚合
  - **如果选 C(标记"已运行-通过")**:`code-dashboard` 统计"已通过"数**虚增**,需更新报告说明
  - **建议**:本需求优先选 A(最低风险)
- **来源**:`./assistants/V0.0.2/require/REQ-00004/RESULT.md` §NFR-8 / §统计字段定义
