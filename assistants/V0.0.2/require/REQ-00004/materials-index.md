> > 增加需求: 添加 `/code-dashboard` 开发看板技能,展示当前版本下需求/任务/缺陷的整体执行情况,并给出一下步的执行建议,如继续未完成的任务或继续未修复的缺陷。不指定需求编号时展示整个版本的总览,以简单图表或进度表的方式展示,指定需求编号时展示需求的进度以及其下各项任务的进度概览。

## 关键信息抽取(供后续步骤使用)

### 1. 技能名
- `/code-dashboard`(对应技能目录 `code-dashboard`,遵循 `skill-conventions.md §规则 1` 的 kebab-case 命名)

### 2. 目标(用户原话拆分)
- 展示**当前激活版本**下:需求/任务/缺陷的整体执行情况
- 给出**下一步执行建议**:
  - 继续未完成的任务
  - 继续未修复的缺陷

### 3. 输入参数
- **不指定需求编号**:整个版本的总览(简单图表或进度表)
- **指定需求编号**:该需求的进度 + 其下各项任务的进度概览

### 4. 隐含约束
- "当前版本" → 读取 `./assistants/.current-version`(与其他 `code-*` 一致)
- "需求/任务/缺陷" → 数据源在 `./assistants/<版本号>/RESULT.md` 的 3 个区段(需求清单 / 任务清单 / 缺陷清单)
- 与现有 `code-version` / `code-require` / `code-design` / `code-plan` / `code-it` / `code-unit` / `code-review` 平级,**新增第 8 个 code-* 技能**

### 5. 不确定项(待澄清)
- "下一步执行建议"的输出形式:纯文本 / 链接到具体命令(如 `建议执行 /code-it REQ-00004-001`) / 两者皆可
- 图表与进度表的形态:ASCII 表格 / Mermaid 图 / 两者皆可
- 缺陷维度是否需要按严重度(P0/P1/P2)分层展示
- "当前版本无激活工作空间"的边界处理(应引导用户先调 `code-version`)
- 是否需要"切回上一版本/查看历史版本"作为附加能力(用户原话未提)
- 多需求(>=3)时,展示是否需要分页/折叠
