# 澄清记录 — REQ-00015

## 2026-06-05 15:50 — 第 1 轮(3 个核心决策)

- **问题 1**:`code-merge` 在遇到冲突时,是「全自动 LLM 智能合并」还是「半自动 — 展示冲突位置 + 停下等用户手动解决」?
- **选项**:
  - A. LLM 智能合并(全自动)
  - B. 半自动:展示冲突+等用户手动
  - C. 混合:有 `--auto` 开关
- **用户回答**:A(LLM 智能合并,全自动)
- **影响**:RESULT.md §4.3 冲突解决策略;§5 NFR 取消"半自动交互"要求

- **问题 2**:把 worktree 提交合回主分支时,使用哪种 git 手法?
- **选项**:
  - A. `git merge`(默认,保留历史)
  - B. `git merge --squash`(压成单 commit)
  - C. 默认 merge,可加 `--squash` 切换
- **用户回答**:A(`git merge` 默认,保留历史)
- **影响**:RESULT.md §4.5 合并手法

- **问题 3**:SKILL.md 算「过程文件/结果文件」吗?
- **用户回答**:"合并代码一般应该不会产生额外的 SKILL.md 文件,所以不需要专门针对 SKILL.md 的策略。"
- **影响**:此回答未直接回答"首次创建时是否必须产 SKILL.md",需二次澄清(见第 2 轮)

## 2026-06-05 15:50 — 第 2 轮(2 个关键澄清)

- **问题 4**(二次澄清):首次创建 `code-merge` 技能时,SKILL.md 是否必须产?
- **用户回答**:"创建技能当然需要 SKILL.md 文件,这里不产生过程文件和结果文件是技能能力的一部分,要求 code-merge 技能执行过程中不产生额外的文件。"
- **影响**:RESULT.md §4.4 SKILL.md 边界;§NFR-1 限定"code-merge 执行阶段不产任何文件"为技能能力的一部分;SKILL.md 是技能入口,首次创建必产

- **问题 5**:"合并最新的主干分支(可指定)" — 不传主干参数时,默认值是什么?
- **选项**:
  - A. 默认 `main`(本地)
  - B. 默认 `origin/main`(远端最新,自动 git fetch)
  - C. 默认报错,强制用户指定
- **用户回答**:B(默认 `origin/main`,自动 git fetch)
- **影响**:RESULT.md §4.5 主干默认值
