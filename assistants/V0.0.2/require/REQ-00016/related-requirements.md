# 关联需求 — REQ-00016

## REQ-00005(优化 `code-require` / `code-design` / `code-plan`,增加"首步拉取+末步提交")— V0.0.2
- **关联点**:REQ-00005 给 3 个技能加了"首步拉取(步骤 0a)"+ "末步兜底提交(步骤 N)"模式;本需求恰好**正是**"让 2 个技能(`code-design` / `code-plan`)在有些项目中能跳过非必要步骤 + 末尾提交无需用户确认"
- **影响**:本需求是 REQ-00005 的**自然延伸**——REQ-00005 加了"末尾兜底提交"但保持 3 选 1 确认;本需求**进一步**让 2 个技能(`code-design` / `code-plan`)的"末步兜底提交"在特定项目类型下(用户显式指定)跳过确认直接 commit
- **版本**:V0.0.2

## REQ-00007(`code-auto` 自动开发技能)— V0.0.2
- **关联点**:`code-auto` 已经在内部强制"所有 `AskUserQuestion` 自动选推荐项"(FR-6 + Q-4 锁定 A);而 `code-design` / `code-plan` 步骤 N 的"3 选 1 确认"恰好是 `AskUserQuestion` 的典型场景
- **影响**:`code-auto` 用户已隐含"全自动"诉求;本需求让 2 个技能**默认**走"无需确认直接提交"路径(在 `code-auto` 场景下);而用户在主流程单独调用 `code-design` / `code-plan` 时,仍可**显式选择**"需要 3 选 1 确认"或"不需要确认直接提交"
- **版本**:V0.0.2

## REQ-00008(`code-review` 整版本模式)— V0.0.2
- **关联点**:REQ-00008 让 `code-review` 增加"无参模式"= 整版本评审,本身就是"模式扩展"的范例
- **影响**:本需求延续 REQ-00008 的"模式扩展"思路——为 `code-design` / `code-plan` 增加"快模式"和"完整模式"两个入口(快模式跳过非必要步骤 + 跳过非必要过程文档 + 末尾提交无需确认;完整模式保留现有全部行为)
- **版本**:V0.0.2

## REQ-00014(`code-plan` 任务拆分维度:按功能点拆)— V0.0.2
- **关联点**:REQ-00014 优化了 `code-plan` 的"按功能点拆"规则;本需求让 `code-plan` **进一步**支持"快模式"
- **影响**:本需求**不**修改 REQ-00014 既有规则;快模式下 `code-plan` 仍按"按功能点拆",只是**减少**部分过程文档(不写 7 份过程文档,只写 2 份核心 RESULT.md + PLAN.md)
- **版本**:V0.0.2

## REQ-00015(新增 `/code-merge` 技能)— V0.0.2
- **关联点**:REQ-00015 的 NFR-1 强约束"不产生过程文件 / 结果文件(执行阶段)"与本需求"跳过非必要过程文档"语义**部分重合**——都是"减少不必要输出"
- **影响**:本需求**不**触发 REQ-00015 升级;但本需求是 `code-merge` 思想的延续(都在优化"过程文档生成粒度")
- **版本**:V0.0.2
