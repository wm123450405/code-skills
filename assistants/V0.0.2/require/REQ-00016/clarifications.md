## 2026-06-05 16:05 — 第 1 轮(2 个核心决策)

### 问题 1:`code-design` / `code-plan` 的"快模式"如何触发?

- **问题**:用户希望"部分步骤在有些项目中没有必要执行,针对没有必要执行步骤直接跳过",那"快模式"应该用哪种方式触发?
- **选项**:
  - A. 全局开关(环境变量 `CODE_FAST_MODE=1` 永久开启)
  - B. 单次开关(`/code-design --fast REQ-NNNNN` CLI 标志)
  - C. 双模式并存(`/code-design-fast REQ-NNNNN` 新技能入口)
  - D. 自动推断(根据项目类型/规模自动决定)
- **用户回答**:A + B 组合(全局开关作为默认行为 + 单次 CLI 标志可覆盖)
  - 默认行为:环境变量 `CODE_FAST_MODE=1` 时 2 个技能**默认走快模式**;未设置时**默认走完整模式**(保留现有全部行为)
  - 单次覆盖:`/code-design --fast REQ-NNNNN` / `/code-plan --fast REQ-NNNNN` / `/code-design --full REQ-NNNNN` / `/code-plan --full REQ-NNNNN` 显式指定
- **影响**:RESULT.md §FR-1 (快模式入口)+ §FR-2 (快模式跳过哪些步骤)+ §NFR-1 (默认行为);`code-design` / `code-plan` SKILL.md 改动 = "新增 §步骤 0a.5 模式选择"

### 问题 2:`code-design` / `code-plan` 的"末尾兜底提交"在快模式下应该如何处理?

- **问题**:用户希望"对于最后阶段的提交代码步骤要确保生成的过程文档也要一并提交,提交过程无需用户确认,直接提交就可以了"
- **选项**:
  - A. 快模式**完全跳过** 3 选 1 确认,直接 `git commit`(如果 dirty)
  - B. 快模式仍然弹 3 选 1 确认(仅做提示,默认 A)
  - C. 快模式**完全跳过** 步骤 N 末尾兜底提交(留给用户手动)
- **用户回答**:A(快模式跳过 3 选 1 确认,直接 `git commit`)+ 关键约束"确保生成的过程文档也要一并提交"——意思是:`git add` 必须把所有**过程文档 + 结果文件**都 add 进去,**不**遗漏过程文档
- **影响**:RESULT.md §FR-3 (快模式末尾提交行为)+ §NFR-2 (过程文档必含);`code-design` / `code-plan` SKILL.md 改动 = 步骤 N 步骤 4"3 选 1 确认"前增加判断"如果模式=快,跳过此步直接进入步骤 5"
