# FAQ — V0.0.5

> 本文件聚合 V0.0.5 版本常见问题,供查询导出使用。

## Q1: V0.0.5 在 v2 简化版看板体系下做了什么?

V0.0.5 完成了 REQ-00044 重构后丢失能力的回补,新增 5 条需求(REQ-00045~49)并修复 8 条缺陷(BUG-00001~08),核心产出包括 6 个语言适配文件、5 个 references 增强、`--confirm` 模式落地、工作流门控机制。

## Q2: --confirm 模式如何使用?

```bash
/code req "<需求描述>" --confirm   # 每阶段完成后强制确认
/code fix "<缺陷描述>" --confirm
```

`--confirm` 与 `--auto` 互斥,二者不可同时指定。

## Q3: 看板上"待需求分析"如何判定?

需求最后阶段 = `INIT` 时归入"待需求分析";缺陷不经过此状态。

## Q4: PROCESS.md 缺失会怎样?

视为 0 阶段完成,看板按缺失前的最后阶段计入。

## Q5: 如何跨版本查询需求?

```bash
/code faq "<关键词>"             # 跨版本搜索
/code faq --require REQ-XXXXX ./out.md  # 导出需求文档
```

## Q6: 发布检查会生成哪些文件?

- `assistants/<版本>/publish/DEPLOY.md` — 部署手册(始终)
- `assistants/<版本>/publish/UPDATE.md` — 更新日志(非基线时)
- `assistants/faq/FAQ.md` — 本文件(始终)

## Q7: 切换版本前需要做什么?

若当前版本有未发布内容,系统会询问"是否先发布"。本仓库已实施"先发布再切换"的友好提示。

## Q8: BUG-00005 / BUG-00006 为何未在主看板缺陷清单登记?

发布检查发现看板存在数据缺口(实际目录含 8 条 BUG,看板仅登记 6 条);本次发布在 `DEPLOY.md` 验证清单中显式声明这 2 条已纳入发布范围,后续 `code-fix` 应将其补入看板。
