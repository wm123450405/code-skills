# 缺陷分析 — BUG-00003 · code-req 技能被调用时跳过工作流直接修改代码

> 所属版本:V0.0.5
> 创建时间:2026-06-30 21:40

## 1. 缺陷描述

`/code-req` 技能被调用时,AI 多次跳过既定的 REQUIRE→DESIGN→PLAN→CODING→CHECK→DONE 工作流,直接进入代码修改。从 CONTEXT.md 中可以看到两次违规:

1. **"Q&A 改为 FAQ"任务**(未加 --auto):直接编辑了 code-ver/SKILL.md、code-ver/references/common.md 等文件,未创建 REQUIRE.md/DESIGN.md/PLAN.md/PROCESS.md
2. **"修改技能描述"任务**(未加 --auto):直接编辑了 7 个 SKILL.md 的 description 字段,未创建任何工作产物

**严重程度**:高(违反技能核心契约)

## 2. 触发条件

1. 用户调用 `/code-req <需求描述>` 无 `--auto`
2. AI 模型将任务判定为"简单修改"而非"需求开发"
3. AI 直接使用 Edit/Write 工具修改 CWD 源码,跳过工作流

## 3. 可能成因

### 根因: SKILL.md 中工作流强制执行指令位置不够靠前

SKILL.md 的结构是:
```
---
frontmatter
---
# 标题
## 目标
## 适用场景
## 不适用
## 工作目录约定(强制)
## 输入
## 输出
## 工具使用约定
---
## 工作流程
  ### 强制阶段门控(最高优先级)  ← 离文档开头约 80 行
  ### 步骤 0 — 版本检测 + 恢复执行
```

AI 在读取 SKILL.md 时,前面的"目标""适用场景""不适用""输入""输出""工具使用约定"等章节可能让 AI 形成"直接修改文件是合理的"的判断,而不等到读到"强制阶段门控"就已经开始执行。

**关键问题**:工作流强制指令被放置在文档中后部,前面约 80 行的内容稀释了其强制性。

## 4. 影响范围

- `code-req/SKILL.md`:需将工作流强制指令移至文档最前面(frontmatter 之后)
- `code-fix/SKILL.md`:同样需要前置工作流强制指令
