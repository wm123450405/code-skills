# 修复设计 — BUG-00003 · 工作流强制指令位置

> 所属版本:V0.0.5
> 创建时间:2026-06-30 21:42

## 1. 设计概述

将 `code-req/SKILL.md` 和 `code-fix/SKILL.md` 的**工作流强制指令移到文档最前面**(frontmatter 之后、标题之后的第一段),确保 AI 在任何情况下都先读到工作流强制要求,不被前面其他章节稀释。

## 2. 改修方案

### 2.1 SKILL.md 结构重组

**当前结构**(问题):
```
---
frontmatter
---
# 标题
## 目标
## 适用场景
## 不适用
## 工作目录约定
## 输入
## 输出
## 工具使用约定
---
## 工作流程         ← 太靠后,AI 可能在此之前已形成跳过工作流的判断
  ### 强制阶段门控
  ### 步骤 0
```

**新结构**:
```
---
frontmatter
---
# 标题

> **⚠️ 强制工作流:本技能必须按以下阶段顺序执行,严禁跳过或合并任何阶段。**
> **INIT → REQUIRE → DESIGN → PLAN → CODING → CHECK → DONE**
> **在 PROCESS.md 显示 CODING 阶段之前,严禁修改任何 CWD 源码文件。**

## 目标
## 适用场景
...
```

### 2.2 新增"启动检查"步骤

在 SKILL.md 的"步骤 0"之前,增加一个"启动检查"段,作为 AI 读取 SKILL.md 后的**第一个操作**:

```
## 启动检查(读本文件后立即执行)

1. 读取 `./assistants/.current-version`,不存在 → 停止,提示调 `code-ver`
2. 解析用户输入,分配需求编号
3. 创建 `req/<REQ-NNNNN>/` 目录
4. 初始化 `PROCESS.md`(追加 INIT 开始行)
5. 开始执行 REQUIRE 阶段
```
