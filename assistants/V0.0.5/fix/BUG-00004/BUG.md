# 缺陷分析 — BUG-00004 · code-req DONE 阶段兜底提交未触发

> 所属版本:V0.0.5
> 创建时间:2026-06-30 22:30

## 1. 缺陷描述

`code-req` 技能在 DONE 阶段完成后,兜底提交代码的动作没有触发。从 CONTEXT.md 中可以看到多个需求/缺陷在 DONE 阶段后均未执行 git commit。

**严重程度**:中(代码变更未提交,用户需手动 git commit)

## 2. 触发条件

1. `code-req` 或 `code-fix` 执行到 DONE 阶段
2. AI 完成"追加 PROCESS.md"和"屏幕输出报告"后,跳过兜底提交步骤
3. 代码变更保持在工作区未提交状态

## 3. 可能成因

### 根因: DONE 阶段描述中兜底提交指令不够显式

当前 DONE 阶段描述:
```
### 步骤 6 — DONE(收尾)
- 追加 PROCESS.md ...
- 屏幕输出完成报告...
- 兜底提交代码(详见 references/common.md §10):  ← 引用形式,AI 可能跳过
  - 非 git 仓库 → 跳过
  - ...
```

`common.md §10` 中有详细的 5 步提交流程,但 DONE 阶段仅以"详见 references/common.md §10"引用,AI 可能将其视为"参考信息"而非"必须执行的步骤",在完成前两个 bullet 后即认为阶段完成。

## 4. 影响范围

- `code-req/SKILL.md` §步骤 6:需将兜底提交改为显式指令
- `code-fix/SKILL.md` §步骤 6:同上
