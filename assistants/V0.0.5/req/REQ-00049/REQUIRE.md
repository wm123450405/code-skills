# 需求分析 — REQ-00049 · 为 code-req、code-fix 增加 --confirm 模式

> 所属版本:V0.0.5
> 创建时间:2026-06-30

## 0. 用户原始输入

> 为 `code-req`、`code-fix` 技能增加 `--confirm` 模式：启用 `--confirm` 模式后每个阶段在进入下一个阶段时都需要强制进行用户确认环节（继续/中止），用户确认时允许用户手动修改产出物的内容，用户确认时提示即将结束阶段的产出物文件路径，用户确认继续后重新读取上一阶段产出物内容（获取用户最新修改的内容）并继续后续阶段。未启用 `--confirm` 模式时自动进入下一个阶段，用户无需确认各阶段产出物。

## 1. 需求概述

为 `code-req` 和 `code-fix` 技能新增 `--confirm` 命令行参数。启用后，每个阶段完成后强制弹出用户确认环节，提示产出物文件路径，允许用户手动修改产出物，确认继续后重新读取产出物以获取用户修改。未启用 `--confirm` 时，阶段间自动流转，无需用户确认。

## 2. 背景与目标

### 背景

当前 `code-req` 和 `code-fix` 的阶段间确认行为：
- 默认(无 flag)：每个阶段完成后弹出 `AskUserQuestion`(继续/暂停/取消)
- `--auto`：跳过所有 `AskUserQuestion`，自动继续

问题：
- 默认确认环节过于简单，不提示产出物路径，用户不知道哪些文件被修改
- 用户无法在确认时手动修改产出物内容
- 确认后不重新读取产出物，用户手动修改可能被覆盖

### 目标

1. 新增 `--confirm` 模式，提供增强的阶段确认体验
2. 简化默认行为：无 flag 时自动流转，无需确认
3. `--confirm` 与 `--auto` 互斥

## 3. 功能需求(FR)

### FR-1:新增 --confirm 命令行参数
- **描述**:在 `code-req` 和 `code-fix` 的输入参数中新增 `--confirm` 可选标志
- **输入**:`--confirm`(可选)
- **输出**:激活增强确认模式
- **来源**:用户原始输入

### FR-2:--confirm 模式阶段确认
- **描述**:启用 `--confirm` 后，每个阶段完成后弹出确认环节
- **确认内容**:
  - 提示"即将结束阶段"的产出物文件路径列表(绝对或相对路径)
  - 选项:A.继续(用户可先手动修改文件,确认后重新读取) / B.中止
  - 不提供"暂停"选项(--confirm 模式下要么继续要么中止)
- **来源**:用户原始输入

### FR-3:产出物文件路径提示
- **描述**:确认时显示刚完成阶段的所有产出物文件路径
- **路径提示格式**:
  ```
  阶段 <阶段名> 完成,产出物:
    - <产出物1路径>
    - <产出物2路径>
  
  你可以手动修改上述文件,完成后选择继续。
  ```
- **来源**:用户原始输入

### FR-4:确认后重新读取产出物
- **描述**:用户确认继续后，必须重新读取上一阶段的产出物内容，以获取用户手动修改的最新内容
- **重读范围**:阶段产出物(REQUIRE.md/DESIGN.md/PLAN.md 等核心文档)
- **来源**:用户原始输入

### FR-5:默认行为变更
- **描述**:未启用 `--confirm` 时，阶段间自动流转，不弹出确认
- **变更**:当前默认行为(弹出 AskUserQuestion)改为自动流转
- **来源**:用户原始输入

### FR-6:--confirm 与 --auto 互斥
- **描述**:`--confirm` 和 `--auto` 不能同时使用
- **冲突处理**:同时传入时，提示错误并退出
- **来源**:设计推断(两个模式语义互斥)

### FR-7:code-fix 同步支持
- **描述**:`code-fix` 技能同步支持 `--confirm` 模式，行为与 `code-req` 一致
- **来源**:用户原始输入
