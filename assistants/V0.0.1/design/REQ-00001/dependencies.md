# 三方依赖评估 — REQ-00001
更新时间:2026-06-03 20:25
版本:V0.0.1

## 结论

**本需求不新增任何第三方依赖**。本仓库是 Claude Code 技能定义集合,无应用代码、无构建工具、无运行时依赖;新增的"工具"是字符串编辑本身。

## 现有依赖盘点

| 类别 | 数量 | 说明 |
| --- | --- | --- |
| 第三方运行时依赖 | 0 | 本仓库无 Node.js / Python / Go 等代码 |
| 第三方构建工具 | 0 | 本仓库无 package.json / requirements.txt / go.mod 等 |
| 第三方测试框架 | 0 | 本仓库无任何测试代码 |
| 第三方 Lint 工具 | 0 | 本仓库无任何 Lint 配置 |
| Claude Code 协议依赖 | 2(隐式) | `marketplace.json` / `plugin.json` 的 `$schema` 引用官方 JSON Schema;本需求不改 `$schema` 字段 |

## 评估本需求是否需要新增依赖

| 候选工具 | 评估 | 结论 |
| --- | --- | --- |
| 自动化字符串替换脚本(shell / sed / awk) | 不必要 — `Edit` 工具已能精确替换;`Grep` 工具已能验证;引入脚本违反 NFR-5(最小化) | **不引入** |
| JSON 校验工具(jq / jsonlint) | 不必要 — Claude Code 协议清单已在 Claude Code 加载时被解析,加载失败会立即报错;`code-review` 阶段 diff 审阅即可 | **不引入** |

| CI 流水线(改名后跑 lint) | 不必要 — 本仓库无 CI 配置,且 NFR-5 显式禁止引入 | **不引入** |

## 决策依据

- `NFR-5(可维护性 / 最小化)` 显式要求:"不引入任何额外抽象、配置项、参数化命名机制;不为本需求新增工具脚本、CI 配置或自动化检查"
- `FR-7` 显式要求"本需求严禁修改任何 SKILL.md / 模板 / 规范文件",也间接禁止"为支持本需求而新增工具"
- `module-conventions.md §规则 1` 仅约束资源摆放,不约束依赖管理(本仓库无依赖管理需求)
