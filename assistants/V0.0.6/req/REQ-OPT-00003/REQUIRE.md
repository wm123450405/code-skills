# 需求分析 — REQ-OPT-00003 · 移除板块标题序号,引用处同步改为板块标题

> 所属版本:V0.0.6
> 创建时间:2026-07-24 11:15

## 0. 用户原始输入

> 移除所有技能、模板等文件中各个板块标题中的序号,以及所有文件中引用这些板块内容时也要修改使用板块标题而非序号。

## 1. 需求概述

清理 `plugins/code-skills/` 下所有 .md 文件中**板块标题**里的序号(包括 `1.` `1.2` `§1` `§2.1` `9a` `5a-1` 等),并把所有引用这些板块的"§N"或"N"形式同步改为**板块标题文字**。

## 2. 背景与目标

- **背景**:为方便用户阅读与维护,板块标题不再依赖序号定位,引用改为板块标题语义,板块增删改时引用不必同步改数字
- **目标**:板块标题一律是文字;正文引用一律是文字;保留板块的层级关系(`##`/`###`/`####`)

## 3. 功能需求(FR)

### FR-1 板块标题去序号

所有 `##` / `###` / `####` 标题若以 `\d+` 或 `\d+(\.\d+)*` 或 `§\d+` 或 `§\d+(\.\d+)*` 开头,移除序号与前缀(含 `.`、` `、`§` 等),保留标题文字。

### FR-2 正文中 §-引用改为板块标题

正文中所有 `§N` `§N.N` `§Na` `§N-N` `DESIGN.md §N` `BUG.md §N` 等格式,改为对应的**板块标题文字**。

### FR-3 跨文件引用与本文件标题保持一致

跨文件 `详见 common.md §N`、`参见 fix-register.md §N` 等,改为板块标题;若无法唯一确定(同名多板块),按上下文选最相关的一个。

## 4. 非功能需求(NFR)

### NFR-1 范围限定

仅清理 `plugins/code-skills/skills/code/` 下的 .md 文件;不清理:
- README.md / README.en.md 顶部的 `# 1./2./3.`(用户明确)
- FR-/NFR-/AC-/E-/INV- 等设计编号
- TASK-/BUG-/REQ- 编号、commit scope

### NFR-2 板块标题去重保护

若清理后两个标题相同,**保留后者的层级语义**(不强行加序号,改用描述性短语消除歧义)。

### NFR-3 不改 frontmatter

`.md` 文件顶部 YAML frontmatter(`---` 块)内的字段值**不**视作板块标题,即使形如"FR-1"也不动。

### NFR-4 不改代码块/引用块/表格

代码块(```)、引用块(`>`)、表格行内的序号**不**清理(它们是代码/示例内容,非文档结构)。

## 5. 验收标准(AC)

### AC-1 板块标题无序号
- 全仓库扫描:`^#+\s+\d+(\.\d+)*\s` 命中数 = 0
- 全仓库扫描:`^#+\s+§\d` 命中数 = 0

### AC-2 板块标题文字保留
清理后每个原板块标题的**文字部分**仍在,层级(`##`/`###`/`####`)不变。

### AC-3 正文 § 引用全部改为文字
- 全仓库扫描:`§\d+(\.\d+)*` 在正文(非标题行)命中数 = 0
- 例外:已被本 REQ 改为文字的引用

### AC-4 README/设计编号未误伤
- `README.md` / `README.en.md` 顶部 `# 1./2./3.` 仍存在
- `FR-1` / `NFR-1` / `AC-1` / `E-M1` / `INV-9` 等设计编号仍存在

### AC-5 无残留歧义
清理后任意两个同名标题(同层级)仅出现 1 次;若有冲突,按 NFR-2 解决。
