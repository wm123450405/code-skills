# 需求分析 — REQ-OPT-00002 · 技能"不要做"事项整治 + 全技能精简单调

> 所属版本:V0.0.6
> 创建时间:2026-07-22 12:15
> 模式:默认
> 范围:主 SKILL.md + 7 个子命令 SKILL.md(references/{ver,req,fix,faq,rule,merge,help}/SKILL.md)

## 0. 用户原始输入

> 用户输入的命令:
> `/code:code req 检查主技能文件与所有子命令技能文件,找到所有强调不要做的事项,组条分析每条事项分别做以下处理:
> 1. 分析不要做的事项是否已是技能执行过程中环节,若已在技能执行过程中有相关内容,应该移除该条不要做事项并在对应的执行内容上增加必须性或重要性强调;
> 2. 若不要做事项是额外补充的内容,分析是否能够将语法修改为必须做事项(不能是"必须不做"类型),设立必须做事项清单,避免出现强调不要做的具体内容;
> 3. 若即不是技能执行过程中内容,也无法直接修改为必须做类事项则保持在现有不要做事项清单中。
> 以上三点完成后,最后检查所有技能内容,精简繁复的描述(必须能够保留原有意思)以及移除空白或不必要的内容(对技能本身执行没有有帮助的内容)。`

## 1. 需求概述

本 REQ 对 8 个 SKILL.md 中的所有"不要做"事项做一次梳理整治,目标:
1. 把与现有流程/不变式重复的"不要做"项移除,改在原强制流程处加"必须性/重要性"强调
2. 把可重写为"必须做"的事项改写,新设"必须做事项清单"
3. 保留确实无法转化的少数项
4. 整治后做一次全技能精简单调(消除冗余表述,移除无帮助内容)

## 2. 背景与目标

### 背景

- 本仓库 8 个 SKILL.md 共统计 95 条"不要做"事项(主 6 + ver 13 + req 23 + fix 18 + faq 8 + rule 9 + merge 18)
- 大量"不要做"项与 §0 不变式 / common.md 阶段执行器 / 各契约层 / 各子命令 SKILL.md 的"启动纪律自检表"重复
- 主 SKILL.md 描述 opt.md §3.14(P2)指出"§0 自检会在每次调用时打断用户";但更深的问题是:同一规则在多处复述(`不要做`项 + 不变式 + 启动纪律表 + common.md 阶段执行器),模型/读者读到 4-5 次,内容表达冗余
- opt.md §8.2 明确"不要用更多'不要……'来弥补不清晰的正向流程"

### 目标

1. 95 条"不要做"项,按用户 3 类方法逐一处置,每条都有明确去向
2. 新设"必须做事项清单"(若适用)在各 SKILL.md 内
3. 整治后,每个 SKILL.md 的"不要做"项**不超过** 3 条(只保留确实无法转化的)
4. 全技能精简单调:消除繁复描述 + 移除空白/无用内容,保留原意

## 3. 功能需求(FR)

### FR-1:扫描并分类 95 条"不要做"事项
- **输入**:主 SKILL.md + 7 个子命令 SKILL.md 的"## 不要做的事"段
- **输出**:每条标注去向(类 1 移除 / 类 2 转必须做 / 类 3 保留)
- **分类标准**:
  - 类 1:事项内容已在某处强制(§0 不变式 / 阶段执行器 / 启动纪律自检表 / 契约层 / 子命令不变量),原"不要做"项移除,在原强制处加"必须性/重要性"强调
  - 类 2:可改写为正向"必须做"的(如"不要修改 X" → "修改 X 前必须 Y");新设"必须做事项清单"
  - 类 3:既不在流程中,也无法直接转"必须做",保留在原清单
- **来源**:用户原始输入第 1/2/3 点

### FR-2:类 1 处置 — 移除重复项 + 加必须性
- **输入**:95 条中标注为类 1 的事项
- **输出**:从"不要做"清单移除;在对应流程处(不变式 / 自检表 / 阶段执行器)补充"必须"或"强制"字样
- **示例**:
  - "不要调 EnterPlanMode" → §0 I-2 改为"必须不调 EnterPlanMode(强制,违反即流程失败)"
  - "不要跳过 PROCESS.md 追加" → common.md §3 改为"阶段开始/完成**必须**追加 PROCESS.md 一行"

### FR-3:类 2 处置 — 转"必须做"事项清单
- **输入**:95 条中标注为类 2 的事项
- **输出**:每条改写为"必须做"句式;合并到各 SKILL.md 新设的"必须做事项清单"(位置:`## 不要做的事` 之前)
- **改写规则**:不能是"必须不做"类型(如"必须不做 X" ✗);必须正向(必须 X 前先 Y / 必须 用 Z 而非 W)
- **示例**:
  - "不要修改 RESULT.md 中非本技能负责的区段" → "修改 RESULT.md 时**必须**只动本技能负责的区段,不得越界"
  - "不要重写既有规则文件" → "修改既有规则文件时**必须**用 Edit 在末尾追加;新建分类**必须**用 Write 首次"
  - "不要臆造源文档中不存在的内容" → "导出 FAQ 时**必须**只复制源文档原文,不补充任何额外内容"

### FR-4:类 3 处置 — 保留在原清单
- **输入**:95 条中标注为类 3 的事项
- **输出**:保留在"不要做"清单,但**最多保留 3 条/文件**
- **精简原则**:把同义/近义事项合并为 1 条;若确实无可归并,保留原文

### FR-5:全技能精简单调
- **输入**:整治后的 8 个 SKILL.md + references/{ver,req,fix,faq,rule,merge,help}/ 下的所有 reference
- **输出**:
  - 消除繁复描述(如"严禁 / 不要 / 必须"在同一段重复 3 次以上 → 整合为 1-2 次)
  - 移除空白/无用内容(如"## 衔接" / "## 不要做的事" 等空壳段,内容无信息量)
  - 保留原意(meaning preservation)
- **不**改动的:
  - frontmatter 的 `name` / `description`(子命令 description 是关键触发判定)
  - §0 不变式(强制硬规则)
  - 契约层 `references/_shared/contracts.md`(独立事实源)
  - opt.md 已锁定的对外行为契约

### FR-6:整治后可量化指标
- **指标 1**:每个 SKILL.md 的"## 不要做的事"段 ≤ 3 条
- **指标 2**:每个 SKILL.md 都有"## 必须做事项清单"(若适用)
- **指标 3**:总"不要做"项从 95 条降到 ≤ 30 条
- **指标 4**:重复内容(同义出现在 2 处及以上)从可测量降到 0
