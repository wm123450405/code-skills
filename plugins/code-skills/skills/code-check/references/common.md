# 通用流程细节 — code-check

> 本文件为 code-check 技能提供语言无关的通用流程细节。始终加载。

## §1 步骤 0 — 版本上下文检测
Read .current-version → 不存在即中止

## §2 步骤 1.5 — 模式选择
- 无参 → 整版本模式
- REQ-NNNNN → 单需求模式
- BUG-NNNNN → 单缺陷模式

## §3 步骤 2 — 整版本模式
1. 过滤需求(状态=已完成)
2. 循环评审(复用模式 1 步骤 4-15)
3. 写聚合 REVIEW.md
4. 派生任务追加
5. 报告

## §4 步骤 4 — 读取上游文档
依次:require/RESULT.md → design/RESULT.md → plan/RESULT.md → plan/PLAN.md

## §5 步骤 5 — 列出待评审任务
筛选条件:开发状态 ∈ {已完成, 进行中} ∧ 测试状态 ∈ {已运行-通过, 已运行-失败, 不适用}

## §6 步骤 6 — 读取代码改修正文
- 读 code/<TASK>/RESULT.md
- 读 code/<TASK>/unit-test-results.md(由 code-it 步骤 8.5 产出)
- 读 CWD 实际源码

## §7 步骤 7 — 加载评审清单
- 优先:./assistants/rules/review-checklist.md
- 兜底:本技能内置 checklists/review-checklist.md

## §8 步骤 8 — 逐任务评审(14 维度)
1. 8.1 正确性(必须改)
2. 8.2 规范遵循(必须改/建议改)
3. 8.3 详细设计符合度(必须改)
4. 8.4 安全(必须改)
5. 8.5 性能(必须改/建议改)
6. 8.6 可维护性(建议改)
7. 8.7 测试质量(必须改/建议改)
8. 8.8 一致性(建议改)
9. 8.9 上下游一致性(必须改)
10. 8.10 详设完整性(必须改)
11. 8.11 概设越界检测(必须改)
12. 8.12 行数比例警告(警告)
13. 8.13 代码行数超标(必须改/建议改/可选)
14. 8.14 过程文档适配性(建议改)

## §9 步骤 9 — 分类发现,决定是否派生
- 必须改 → 派生"审查改修"任务
- 建议改 → 询问用户
- 可选 → 记录到 findings-no-task.md

## §10 步骤 10 — 分配新任务编号并更新 PLAN.md
1. 找到 PLAN.md 当前最大任务序号 N
2. 分配 N+1, N+2, ...
3. 类型=修改,触发/来源=审查改修
4. 追加到 PLAN.md 任务总览 + 任务详情

## §11 步骤 11 — 撰写 review/RESULT.md
按 templates/REVIEW-FIX.md 结构:
1. 任务信息 / 2. 触发的原任务 / 3. 问题清单 / 4. 应当改修的文件 / 5. 验证手段 / 6. 关联依据 / 7. 不需要做的

## §12 步骤 12 — 撰写整体 REVIEW-REPORT.md
按 templates/REVIEW-REPORT.md 结构:
1. 评审信息 / 2. 评审清单 / 3. 任务评审结果总览 / 4. 发现汇总 / 5. 派生任务列表 / 6. 未派生发现 / 7. 超出范围发现 / 8. 整体结论 / 9. 变更记录

## §13 步骤 13 — 同步版本看板
1. 更新任务清单(追加派生任务)
2. 追加评审发现汇总
3. 追加派生任务记录
4. 追加缺陷清单(若有严重缺陷)
5. 追加变更记录

| P3 | 一致性 | 建议改 |
