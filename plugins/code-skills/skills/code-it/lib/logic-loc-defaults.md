# 逻辑行阈值默认值(由 code-check 共用)

- 所属技能:`code-check`
- 创建时间:2026-06-22 15:00
- 适用版本:V0.0.3
- 来源:REQ-00039 详细设计 §4.3(阈值字段)

## 默认值

| 字段 | 默认值 | 说明 |
| --- | --- | --- |
| 单文件逻辑行总规模阈值 | **500** | 单文件修改后的总逻辑行超过此值触发"代码行数超标"发现 |
| 单文件逻辑行新增阈值 | **200** | 单任务对单文件的新增逻辑行超过此值触发"代码行数超标"发现 |

## 阈值级别

| 超标比例 | 级别 |
| --- | --- |
| ≤10% | 可选 |
| >10% 且 ≤50% | 建议改 |
| >50% | 必须改 |

## 用户配置覆盖(FR-5 可选)

若需覆盖默认值,在 `require/<需求编码>/RESULT.md` 的"## 阈值配置"小节中显式配置:

```markdown
## 阈值配置

单文件逻辑行总规模阈值:300
单文件逻辑行新增阈值:100
```

`code-check` 步骤 8.13 读取此配置(若存在)覆盖默认值。

## 不配置时的行为

- 沿用本文件默认值(500 / 200)
- 屏显契约中的"阈值"字段显示沿用的值
- 无额外屏显
