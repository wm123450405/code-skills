# RESULT.md 模板 — 详细设计

> 本文件是 `code-plan` 技能产出的 `RESULT.md` 章节结构与填写规范。
> 模型在生成 `RESULT.md` 时应严格按本结构组织,确保 `PLAN.md` 中每条任务都能追溯到本文件的某个设计点。
>
> **版本感知**:本文件存放在 `./assistants/<版本号>/plan/<需求编号>/RESULT.md`,文档头应明确记录版本号。

## 5. 算法与逻辑(本节**必填**;至少 1 个伪代码块)
**本节是详细设计区别于概要设计的核心:必须给出可直接编码的算法/逻辑**。

```
### 算法 1:<名称>
- **目的**:解决什么问题
- **输入**:(类型与约束)
- **输出**:(类型与含义)
- **复杂度**:时间 O(?),空间 O(?)
- **依赖**:(调用的其他函数/库)
- **伪代码**:
  ```
  function algorithm_1(input):
      validate(input)
      a = step1(input)
      b = step2(a)
      if b is invalid:
          handle_error(...)
      return step3(b)
  ```
- **关键决策与权衡**:为什么用这个算法而不是候选
- **边界条件**:空、极大、极小、并发
- **对应任务**:<任务编号>
- **依据规范**:... §X
```

复杂逻辑用 Mermaid 流程图或状态机图。

## 8. 异常处理(本节**必填**)
按异常类别组织(若某类别不涉及,显式写"本需求不涉及"):
- **输入校验**:参数错误如何提示用户
- **外部依赖**:超时/不可用/限流
- **并发冲突**:锁/事务/重试
- **资源耗尽**:内存/连接/线程
- **业务异常**:业务规则违反
- **未知异常**:全局兜底

每条异常给出:**触发条件 → 检测手段 → 处理策略 → 监控指标 → 对应任务**。

详见 `risk-analysis.md`。
