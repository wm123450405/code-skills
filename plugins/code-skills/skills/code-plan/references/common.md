## §8 步骤 10A — 任务拆分原则
### 核心原则:按"功能点"拆分
- 1 个任务 = 1 个完整功能点 = 1 个用户完整可用的能力
- 完整一套:展示效果 + 功能逻辑 + 使用说明在同一任务
- 可独立验证:每个任务有清晰"完成定义"
- 0.5-2 天内可完成

### 架构任务触发条件
- 触发 1:较高扩展性/可维护性 + 较高复杂度
- 触发 2:涉及未来可能的不同实现方式
- 触发 3:对接不同三方组件
- 触发 4:满足扩展性触发条件
- 触发 5:整体设计目标 = --extensible

### 按设计目标调整任务粒度
| 整体设计目标 | 维度=高 | 至少任务数 | 任务类型 |
| --- | --- | --- | --- |
| --minimal | * | — | 合并同类任务 |
| --balanced | 高/中/低 | — | 默认粒度 |
| --extensible | 扩展性=高 | 1 | 扩展架构设计 |
| --extensible | 封装性=高 | 1 | 封装抽象层 |
| --extensible | 可复用性=高 | 1 | 抽取公共逻辑 |
| --extensible | 可读性=高 | 1 | 关键代码注释 |
