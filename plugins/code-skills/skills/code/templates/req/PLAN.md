## 附录 A:测试状态流转规则(FR-7 来源)

> 状态字面以契约层 `references/_shared/contracts.md`「状态字面表」为唯一事实源。

### A.1 初值表

| 任务类型 | 初始测试状态 |
| --- | --- |
| 代码型(含可执行代码) | 未编写 |
| 文档型(仅 Markdown / 配置文件) | 不适用 |
| 配置型(模板/Schema/契约) | 不适用 |

### A.2 "不适用"判定条件

任务范围不含可执行代码、不需要单元测试或集成测试验证。

### A.3 CODING 阶段状态转移

| 触发事件 | 当前 → 转移后 |
| --- | --- |
| 编译通过 | (无变化) |
| 单测编写完成 | 未编写 → 已编写 |
| 测试运行通过 | 已编写 → 已运行-通过 |
| 测试运行失败 | 已编写 → 已运行-失败 |
| 运行时缺失(经用户授权) | * → 阻塞 |
| 用户明示跳过 | * → 不适用 |

### A.4 CHECK / publish 消费

- `可发布 = 已完成 ∩ {已运行-通过, 不适用}`
- 仅测试状态为 `已运行-通过` 或 `不适用` 的任务视为可发布
- `阻塞` / `已运行-失败` 视为不可发布,阻塞 CHECK 阶段通过
